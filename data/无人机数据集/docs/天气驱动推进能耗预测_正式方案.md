# 固定航线无人机天气驱动推进能耗预测正式方案

## 1. 项目定位

本项目的正式定义为：

在固定无人机型号、固定电池规格、固定载荷、固定航线、固定巡航策略的前提下，利用天气预报或历史气象数据，预测不同气象条件对无人机推进能耗的影响，并在此基础上给出出发前的任务可达性评估。

这个定义有两个关键点：

1. 我们训练的不是天气预报模型，而是续航/能耗预测模型。
2. 我们研究的是天气因素对推进能耗的影响，而不是无人机系统全部机理的全量建模。

## 2. 研究边界

### 2.1 本项目保留的核心问题

- 固定航线下，不同天气条件会导致推进能耗发生怎样的变化。
- 在出发前已知天气预报的情况下，能否预测此次任务的总推进能耗。
- 在固定电池参数的情况下，能否进一步换算出剩余电量、可达性和预警结果。

### 2.2 本项目明确不做的内容

- 不做多机型泛化建模，机型固定为一个实验平台。
- 不做复杂路径规划，航线固定为一条预先设定的路线。
- 不做实时避障与感知建模。
- 不做电池内部电化学机理建模。
- 不做天气预报模型训练，直接使用现成天气数据。
- 不做多无人机协同调度。

### 2.3 关于“路线不考虑”的正式表述

路线不是完全不考虑，而是不作为学习变量自由变化。  
路线、航向、飞行高度、巡航速度都作为实验常量固定下来，只保留它们对天气作用方向判断的意义。尤其是风速和风向，必须结合固定航向才能区分顺风、逆风和侧风。

## 3. 精简后的概念模型

### 3.1 样本定义

一个样本表示：

某固定机型在某固定航线上，于某一出发时间窗内，在给定气象条件下执行一次飞行任务时的推进能耗表现。

### 3.2 输入

输入只分为两类：

1. 固定任务常量  
   这部分不作为重点训练内容，只用于定义场景。
   - 固定机型
   - 固定电池规格
   - 固定载荷
   - 固定航线
   - 固定巡航速度
   - 固定飞行高度

2. 气象特征  
   这部分是模型的核心输入。
   - 风速
   - 风向
   - 阵风
   - 温度
   - 湿度
   - 气压
   - 降水
   - 能见度
   - 天气现象类别

其中最关键的不是原始风向本身，而是相对于固定航线方向计算出来的：

- 逆风分量
- 侧风分量
- 风速强度

### 3.3 输出

模型的主输出建议定义为：

- 总推进能耗

为了让不同任务长度之间更容易比较，也建议同时定义一个标准化输出：

- 单位距离推进能耗

在此基础上，可以通过后处理得到：

- 预计剩余电量
- 是否可完成任务
- 是否触发预警

### 3.4 整体映射关系

项目的核心映射关系为：

```text
气象条件 + 固定任务常量 -> 推进能耗预测
推进能耗预测 + 固定电池参数 -> 可达性评估与预警
```

## 4. 模块划分

为了降低数据集寻找难度，整个系统只保留三个模块。

### 4.1 固定任务配置模块

作用：
定义实验场景，不承担主要学习任务。

内容：
- 无人机型号
- 电池容量
- 额定电压
- 载荷重量
- 巡航速度
- 飞行高度
- 航线长度
- 航线航向角

说明：
这个模块主要是配置表，不是重点数据集模块。

### 4.2 气象特征模块

作用：
收集并整理与飞行任务时间窗对应的天气特征。

核心字段建议：
- 时间戳
- 起点位置
- 终点位置
- 任务时间窗
- 风速
- 风向
- 阵风
- 温度
- 湿度
- 气压
- 降水量
- 能见度
- 天气类型

派生字段建议：
- 逆风分量
- 侧风分量
- 温度区间编码
- 是否降水
- 极端天气标记

说明：
这是项目最重要的数据模块，也是后续寻找公开数据集时的主攻方向。

### 4.3 推进能耗预测与可达性模块

作用：
建立“天气条件到推进能耗”的预测关系，并进一步完成任务可达性判断。

输入：
- 气象特征
- 固定任务常量

输出：
- 总推进能耗
- 单位距离推进能耗
- 预计剩余电量
- 可达/不可达
- 预警结果

说明：
严格来说，真正需要训练的核心只有“推进能耗预测”这一层；可达性和预警层可以通过规则或公式换算实现。

## 5. 数据集需求

本项目不再追求一个“大而全”的无人机综合数据集，而是明确拆成两类数据。

### 5.1 核心数据一：气象数据

这是公开数据最容易获取的部分。

至少需要：
- 时间
- 地点
- 风速
- 风向
- 温度
- 湿度
- 气压
- 降水

优先级最高的字段：
- 风速
- 风向
- 阵风
- 温度

其中最重要的是风相关变量，因为它们最直接影响推进能耗。

### 5.2 核心数据二：推进能耗标签数据

这是最难获取的部分，但也是训练模型必须有的标签。

建议标签形式：
- 单次任务总推进能耗
- 单位距离推进能耗
- 完成任务后的剩余电量

可能来源：
- 自己的小规模实验
- 小车/螺旋桨台架模拟实验
- 简化物理仿真
- 文献中的经验参数

### 5.3 数据获取策略

推荐采用“公开气象数据 + 自采能耗标签”的组合方式。

原因很直接：
- 天气历史数据和天气预报数据容易获得。
- 同一机型、同一航线、同一任务条件下的真实推进能耗公开数据很难直接找到。
- 因此标签部分大概率需要依赖自采实验或仿真补齐。

## 6. 训练样本结构

一个训练样本可以抽象为：

```text
{
  固定机型,
  固定电池,
  固定载荷,
  固定航线,
  出发时间,
  风速,
  风向,
  逆风分量,
  侧风分量,
  温度,
  湿度,
  气压,
  降水,
  阵风
}
-> {
  总推进能耗,
  单位距离推进能耗,
  剩余电量,
  是否可达
}
```

如果要进一步简化，最小可训练版本可以压缩成：

```text
X = {逆风分量, 侧风分量, 温度, 降水}
Y = {单位距离推进能耗}
```

这个版本虽然简化，但非常适合作为前期原型。

## 7. 推荐的项目表达

为了让题目更稳、更聚焦，建议使用以下表述之一：

- 基于气象条件的固定航线无人机推进能耗预测系统
- 面向物流场景的固定航线无人机出发前能耗评估系统
- 基于天气预报的无人机任务可达性预测系统

其中最推荐的表述是：

**基于气象条件的固定航线无人机推进能耗预测系统**

这个题目最贴合你们当前的真实能力边界，也最容易把故事讲圆。

## 8. 当前阶段的结论

当前项目已经可以明确收敛为一条主线：

```text
天气 -> 推进能耗 -> 可达性评估
```

后续所有数据集搜索、模型设计和实验方案，都应该围绕这条主线展开，而不要再回到“多机型、复杂路径、避障、电池机理、天气预测一锅炖”的发散版本。

# 0517 第一视角 3D 无人机展示界面设计建议
建议2D  3D都有
> 本文档用于整理当前项目新增“第一视角 3D 无人机展示可视化模块”的功能建议、页面结构方案与数据接口方向，面向现有任务级预测与回放展示链路，不扩展到低层飞控控制系统。

## 2. 建议增加的核心功能模块

### 2.1 主视角飞行画面区

这是页面核心区域，建议作为中间主画布。

建议功能：

- 第一视角 3D 漫游，沿无人机航线进行机头视角播放
- 视角切换，包括机头视角、跟随视角、俯视总览、自由观察视角
- 三维航线显示，包括已飞轨迹、当前所在点、剩余轨迹、关键航点
- 起点、终点、任务目标点、返航点可视化
- 城市高楼环境建模与障碍物表达
- 禁飞区、限高区、风险区叠加显示
- 建筑净空、离地高度、爬升段、下降段高亮
- 风场、降雨、低能见度等气象图层叠加

这一部分的重点不是特效，而是要让用户一眼看懂“当前飞到哪、后面还怎么飞、为什么这样飞”。

### 2.2 左侧任务态势面板

建议放在页面左侧，用来聚合当前任务关键信息。

建议功能：

- 当前任务名称、任务编号、执行时间
- 无人机型号、载荷配置、电池状态
- 起点、终点、总航程、预计到达时间
- 当前飞行阶段，如起飞、巡航、爬升、下降、接近目标
- 当前环境摘要，包括风速、风向、温度、湿度、天气等级
- 当前预测结果摘要，包括预计耗电、剩余电量、风险评分、返航裕度
- 当前策略解释，如当前路线选择原因、当前速度选择原因

这一块对应“大疆式信息聚合”，但要补上你们项目特有的“预测解释能力”。

### 2.3 右侧分析与告警面板

建议放在页面右侧，用于结果解释和风险提示。

建议功能：

- 实时或回放状态下的速度曲线
- 功率曲线、累计耗电曲线、剩余电量曲线
- 高度曲线、爬升率变化曲线
- 多因素贡献拆解，如风、速度、爬升、绕行、载重对耗电的影响占比
- 风险告警列表，如逆风过强、预计剩余电量不足、进入高风险天气区域、路径接近高楼障碍区
- 当前推荐策略，如保持路线、降低速度、切换备选路线、提高安全高度
- 基线方案与优化方案对比

这一块最容易做出差异化。否则整个系统就只剩下“3D 动画播放器”，价值太虚。

### 2.4 底部时间轴与回放控制

建议作为整个页面底部固定控制区。

建议功能：

- 时间轴拖动
- 播放、暂停、停止、重播
- 倍速播放
- 逐帧查看
- 关键事件节点标记，如起飞、转向、爬升、进入风险区、到达终点
- 异常片段快速跳转
- 不同方案回放切换

这一块要和 `timeseries.csv` 强绑定，不要另起一套完全独立的时序结构。

### 2.5 顶部模式切换与工具栏

建议提供统一的模式切换入口，避免把所有信息硬塞进一个页面。

建议模式：

- 任务总览模式
- 第一视角回放模式
- 三维路线优化展示模式
- 航速变化策略展示模式
- 风险解释模式
- 参数消融与方案对比模式

建议工具：

- 图层开关
- 视角切换
- 时间定位
- 告警筛选
- 结果导出



##  页面布局建议

建议采用以下结构：

- 顶部：模式切换栏 + 工具栏
- 左侧：任务态势与关键指标区
- 中间：第一视角 3D 主画面
- 右侧：曲线分析、风险告警、策略解释区
- 底部：时间轴与回放控制区

建议布局示意如下：

```text
┌──────────────────────────────────────────────────────────────┐
│ 顶部模式栏 / 工具栏                                           │
├───────────────┬──────────────────────────────┬───────────────┤
│ 左侧任务态势区 │        中间第一视角 3D 主画面        │ 右侧分析告警区 │
│               │                              │               │
│               │                              │               │
├───────────────┴──────────────────────────────┴───────────────┤
│ 底部时间轴 / 回放控制 / 关键事件标记                           │
└──────────────────────────────────────────────────────────────┘
```

### 6.2 页面状态建议

建议页面主状态至少包括：

- 当前任务标识
- 当前播放时间索引
- 当前视角模式
- 当前图层开关状态
- 当前选中方案
- 当前选中事件
- 当前告警集合
- 当前任务总览数据
- 当前时刻状态数据

### 6.3 联动原则

建议遵循以下联动逻辑：

- 时间轴移动时，中间 3D 画面同步更新位置和姿态
- 时间轴移动时，左侧任务状态同步更新
- 时间轴移动时，右侧曲线高亮对应时间点
- 点击右侧告警项时，自动跳转到底部对应时间点
- 点击关键事件时，3D 视角自动切换到更合适的观察位
- 切换方案时，3D 航线、指标、曲线、告警全部同步刷新

## 7. 下一步方案二：前端数据接口建议


### 7.1 summary.json 建议承载内容

建议包含：

- 任务基础信息
- 起终点信息
- 总航程、总时长、总耗电
- 预测剩余电量
- 风险总评分
- 方案类型
- 关键事件摘要
- 天气总体摘要
- 推荐策略摘要
- 若存在比较对象，则补充基线方案摘要

### 7.2 timeseries.csv 建议承载内容

建议包含逐时刻字段：

- 时间戳
- 经度、纬度、高度
- 航向角、俯仰角、滚转角
- 速度
- 爬升率
- 功率
- 累计耗电
- 剩余电量
- 风速、风向
- 温度
- 风险分值
- 当前阶段标签
- 事件标签

### 7.3 可选扩展文件

如后续复杂度上升，可增加：

- `alerts.json`
  - 风险告警列表
- `compare_summary.json`
  - 方案对比摘要
- `layers.json`
  - 图层配置与可视化样式配置

但现阶段没必要一次加太多，不然很容易把系统结构搞成一锅粥。

## 8. 下一步方案三：推荐技术方向

### 8.1 三维底座建议

优先建议：

- CesiumJS 作为地理空间 3D 主引擎

原因：

- 适合经纬度、航线、建筑、地形、气象图层
- 更适合城市环境与真实空间关系表达
- 更适合你们“任务级路线展示”这个边界

补充建议：

- 普通页面层继续使用现有 Web 技术栈
- 如果后续要加局部特效或特殊模型动画，再考虑配合 Three.js

### 8.2 展示模式建议

建议至少保留以下几种展示模式：

- 单任务第一视角回放
- 单任务全局 3D 总览
- 两方案对比回放
- 风险热区叠加查看
- 最优路线展示
- 最优航速曲线展示

## 9. 建议的实施顺序

### 阶段 1：最小可用版

- 基于现有 `summary.json + timeseries.csv` 驱动页面
- 完成第一视角 3D 航线回放
- 接入左侧任务状态面板
- 接入底部时间轴
- 接入右侧核心曲线

### 阶段 2：可分析版

- 增加风险告警联动
- 增加天气图层
- 增加策略解释区
- 增加关键事件跳转

### 阶段 3：可比较版

- 增加方案切换
- 增加基线与优化方案对比
- 增加固定路线/固定航速两类优化结果展示

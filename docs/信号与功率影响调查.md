# 信号与功率影响调查

> 本文档基于若干移动通信与无人机通信论文，整理“信号弱或连不到信号时，功率和功耗会发生什么变化”的理论依据。重点回答两个问题：  
> 1. 信号弱时，终端是不是一定会更费电？  
> 2. 连不到信号时，是不是会一直全功率发射寻找，从而导致功耗快速增加？

## 1. 结论摘要

先说结论，不绕弯子：

- `信号变弱` 时，终端的通信功耗通常会升高，是 `码率下降、重传增加、活跃时长变长、tail energy 增加`。
- `连不到信号` 时，终端通常 `不会一直无脑全功率持续发射`，但会进入 `搜网、测量、重选、重连` 等流程，这些动作会带来额外功耗。
- 从“每成功传输一比特数据消耗多少能量”的角度看，`弱信号但仍在勉强连接` 和 `长时间无服务反复重试` 都是高风险状态。
- 对无人机蜂窝联网场景而言，最危险的常常不是“完全静默没网”，而是 `处在弱覆盖边缘、仍在努力维持连接` 的状态，因为这时既有功率抬升，也有重传和时延恶化。


- 弱信号会让 `平均通信功耗` 上升
- 弱信号会让 `单位有效数据能耗` 上升


## 3. 信号弱和功耗的关系

### 3.1 信号弱时，并不只是“发射更用力”

根据 Ding 等人的研究，弱信号会同时触发多层机制变化：

- 物理层会降低调制编码等级
- 传输速率下降
- 链路层和传输层重传增加
- 发送和接收在高功耗状态停留更久
- 发射功率控制可能把上行功率往上推

这意味着：

- 同样一份数据，在弱信号下可能需要更长时间才能传完
- 即使瞬时发射功率不是始终最高，总能耗仍可能明显增加

论文原意非常明确：弱信号不仅影响网络性能，也会显著抬高无线接口的实际能耗。

### 3.2 弱信号会增加“活跃时间”和“尾部能耗”

Huang 等人对 LTE 的研究说明，移动终端能耗不只是由“是否在发包”决定，还强烈受：

- RRC 状态切换
- DRX 行为
- 高功耗 tail 时间

影响。

所以弱信号的麻烦不只是吞吐下降，而是：

- radio 更容易长时间停留在高功耗状态
- 小数据传输也可能拖出较长尾部能耗


### 3.3 已有实验能直接观察到弱信号抬高功耗

Carroll 和 Heiser 的实验中，把设备屏蔽后：

- GPRS 信号强度下降约 `10 dBm`
- GSM 功耗上升约 `30%`

这个结果虽然来自较早期系统，但它非常直观地说明：

- 信号质量下降，本身就可能直接推高无线侧功耗

### 3.4 弱信号下，最差的往往是“勉强在线”

综合这些论文，更稳妥的结论是：

- `完全正常覆盖` 不是高功耗问题
- `完全离线且安静等待` 也不一定是最坏情况
- `弱信号下勉强维持连接` 往往最费电

原因在于这个状态会同时出现：

- 上行功率上调
- 上下行速率下降
- 重传增多
- 时延变大
- 高功耗状态停留更久

也就是说，它兼具“传得费劲”和“传得很久”两个问题。

## 4. 连不到信号和功耗的关系

### 4.1 连不到信号，不等于一直满功率乱发

因此，“连不到信号”通常不是：

- 持续不断地全功率广播

而更像是：

- 周期性地唤醒、监听、测量、尝试连接、失败后再等待下一轮

这点和蓝牙耳机“不断寻找主机”的直觉是相似的，但蜂窝系统更复杂，也更分层。

### 4.2 无服务状态下，额外功耗主要来自搜索与重试

Catovic 等人的论文虽然关注的是 UMTS standby 场景，但它清楚说明：

- UE 在 idle mode 并不是完全睡死
- 它会周期性唤醒
- 需要评估当前小区和邻区信号质量
- 相关调度和睡眠管理会显著影响待机电池寿命

这为下面这个判断提供了理论支撑：

- 连不到信号时，搜网、测量、重选、重连这些行为本身就会增加通信侧功耗

### 4.3 连不到信号时，“单位有效数据能耗”最差

这里有一个很容易被忽视的点：

- 连不到信号时，瞬时发射功率不一定始终最大
- 但有效吞吐可能接近 0

于是从业务视角看，会出现最糟糕的情况：

- 能量已经花出去了
- 但真正有效传回的数据几乎没有

因此：

- 如果只看瞬时发射功率，可能误以为“没有一直拉满，所以问题不大”
- 但如果看 `每成功传输一比特数据消耗的能量`，无服务和反复重试状态往往最差

信号差的情况
```text
接收信号变差 / 路径损耗变大
-> 上行功率需求上升
-> 调制编码降级，链路速率下降
-> 重传次数增加
-> 活跃传输时间变长
-> tail energy 增加
-> 平均通信功耗上升
-> 单位有效数据能耗显著上升
```

如果是“完全连不到信号”

```text
无服务 / 失联
-> 搜网、测量、重选、重连尝试增加
-> 有额外通信侧功耗
-> 有效吞吐接近 0
-> 单位有效数据能耗最差
```

## 6. 无人机场景下为什么这个问题更重要

上面的直接证据主要来自手机或通用蜂窝终端研究，但迁移到无人机蜂窝联网场景是合理的。

原因不是“手机等于无人机”，而是：

- 二者都可能依赖蜂窝网络接入
- 都会受到功率控制、重传、速率自适应、搜索与重连机制影响

而无人机比地面终端更麻烦的一点在于：

- 低空城市环境里更容易经历遮挡切换
- 更容易看到多个强邻区，干扰更复杂
- 基站天线下倾导致高度变化时覆盖不稳定
- 空中运动让切换、姿态变化、链路波动更频繁

因此，无人机蜂窝链路更容易进入：

- `弱信号但仍连接`
- `时断时续`
- `频繁重连`

这些正是通信功耗和链路风险都偏高的状态。


## 8. 当前可采用的正式结论

基于这些论文，可以给出一个相对稳妥的正式结论：

- 信号弱时，终端的通信功耗通常会上升，
- 更关键的机制是：速率下降、重传增加、状态停留变长、tail energy 增加
- 连不到信号时，终端通常不会一直无脑全功率持续发射，但搜网、测量、重选和重连会引入额外功耗
- 从单位有效数据能耗角度看，弱信号边缘连接和反复失联重试都是高风险状态
- 这些关系迁移到无人机蜂窝联网场景是合理的，且在城市低空环境中问题可能更突出

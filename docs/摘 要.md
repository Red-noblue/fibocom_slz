基于 SC171V3 在天气因素与城市环境下无人机里程与能耗的预测与规划系统技术报告

# 摘 要

针对城市低空无人机在高楼密集区运行时容易受到阵风、逆风、侧风、温度变化和建筑遮挡影响，导致续航估计偏差增大、航线安全裕度不足等问题，本作品设计了一套面向任务执行前评估与半实物验证的无人机里程与能耗预测规划系统。系统以 SC171V3 边缘智能模组为核心，采用“边缘端开发板 + PC/云端算法与仿真资源”的分层协同架构。PC/云端负责获取气象数据、重建低空垂直天气剖面、组织城市建筑数据并生成候选三维航线；SC171V3 端负责接收任务参数、执行本地能耗估计、显示关键状态，并联动板端传感采集与蜂鸣器告警。系统在算法上结合空气动力学机理计算和轻量化残差修正模型，对风速、风向、温度、载荷、电池状态和城市街谷环境带来的能耗变化进行综合估算；在规划上，将路径长度、逆风代价、侧风风险、爬升开销、建筑安全距离和剩余电量约束共同纳入搜索过程，输出可解释的候选航线与风险结论。为降低真实试飞风险，系统采用固定式采集、显示与告警验证样机，通过温湿度气压一体传感器、风速风向传感器和电压传感器采集现场状态，并通过显示屏和蜂鸣器验证风险提示逻辑。配套 Web 工作台用于展示真实城市建筑、天气采样点、航线、禁飞区和资产纠错结果，使验证环境具备可检查、可复现和可演示的特点。

# 第一部分 作品概述

## 1.1 功能与特性

本系统面向城市低空无人机任务，完成起终点配置、低空天气剖面生成、任务能耗预测、候选航线规划、风险告警和三维虚拟验证等功能。系统不替代底层飞控，而是位于任务决策层，在起飞前给出剩余里程、预计能耗、推荐航线和风险等级；在半实物样机中，通过温湿度气压一体传感器、风速风向传感器、电压传感器、显示屏和蜂鸣器验证边缘端状态判断与告警结果。其主要特性是边云协同、可解释能耗建模、三维城市约束规划、板端实时采集和可复核的 Web 验证界面。

## 1.2 应用领域

系统适用于城市低空物流、园区巡检、应急勘察、电力与管网巡检等任务。此类任务通常运行在高楼、桥梁、道路和开阔水面混合分布的环境中，气象条件和障碍物分布会直接影响耗电速度和安全航线选择。本作品可在任务前对不同高度层、不同航向和不同候选航线进行比较，为是否起飞、采用哪条航线、剩余电量是否足够提供依据。对于竞赛展示和教学实验场景，固定式采集告警样机可在室内复现风场突变、供电状态变化和告警触发过程，避免真实飞行带来的安全和场地限制。

## 1.3 主要技术特点

系统的技术路线强调工程闭环。第一，能耗估计不只依赖电量线性外推，而是把相对空速、逆风分量、空气密度、爬升速度和电池温度引入计算，再用轻量化模型补偿城市局地风场和样机采集误差。第二，路径规划采用三维空间搜索，节点代价同时考虑距离、风场、建筑净空和剩余电量，使结果不是单纯最短路，而是面向任务可达性的低耗安全路。第三，SC171V3 端保留本地显示、告警和外设联动能力，在上位机离线或网络波动时仍能执行基本判断。第四，Web 工作台把城市、天气、航线和纠错结果放到统一空间坐标中，便于检查验证输入是否合理。

# 第二部分 系统组成及功能说明

## 2.1 整体介绍

系统从工程上划分为边缘端、算法服务端、三维虚拟验证端和板端采集告警样机四个部分。边缘端以 SC171V3 开发板为核心，承担任务配置接收、预测结果显示、传感器实时采集、本地阈值判断、蜂鸣器告警和日志回传；算法服务端部署在 PC 或云端，负责天气数据融合、垂直气象剖面重建、三维候选航线生成和模型训练；三维虚拟验证端负责把城市建筑、地面要素、禁飞区、天气采样和航线结果统一显示在 Web 工作台中；板端采集告警样机通过温湿度气压一体传感器、风速风向传感器、电压传感器、多电源模块、显示屏和蜂鸣器验证系统在现场扰动条件下的响应。需要说明的是，前端页面当前使用的气象数据仍为历史气象样本或离线接口产物，板端传感器用于现场实时测量和本地告警。

整体工作过程为：用户首先在工作台或边缘端输入起点、终点、飞行高度、载荷、电池容量和任务时间等参数；系统从气象模块获取对应位置和高度层的风速、风向、温度、气压与空气密度，并通过城市数据获得建筑边界和安全净空限制；路径规划模块依据天气和建筑约束生成若干候选航线；能耗预测模块计算每条航线的预计耗电量、剩余电量和风险等级；最后，结果被送入 Web 工作台进行空间复核，并可同步到 SC171V3 样机执行页面显示、传感器状态更新和蜂鸣器告警。实物中的电源模块主要承担外设供电职责，不作为无人机飞行电池实物模块描述。

该架构的重点不是把所有计算都放到单一设备上，而是在实时性、算力和可展示性之间做分工。云端或 PC 端适合完成城市数据处理、天气场重建和全局搜索；SC171V3 端适合完成快速判断、状态展示和外设控制。通过 HTTP、串口或文件交换，系统能够在保持边缘端可运行的同时，利用上位机完成更高维度的仿真验证。

## 2.2 硬件系统介绍

硬件系统采用桌面级半实物在环方案。该方案不直接驱动真实无人机飞行，而是用固定式采集、显示与告警验证结构复现无人机任务前评估中的环境测量和风险提示过程。系统通过温湿度气压一体传感器获取温度、湿度和气压，通过风速风向传感器获取现场风场，通过电压传感器获取供电状态，并由显示屏和蜂鸣器呈现边缘端判断结果。这样既可以保留嵌入式系统的采集、通信、控制和显示链路，又能够避免室外试飞对场地、人员和安全防护的要求。

### 2.2.1 硬件整体介绍

硬件总体由主控核心、传感采集端、蜂鸣器告警端、供电端和显示端组成。主控核心选用 SC171V3 开发板，负责解析上位机下发的任务参数和预测曲线，读取温湿度气压一体传感器、风速风向传感器和电压传感器的实时数据，并按照当前风场、气象、供电状态和任务状态计算风险等级。传感采集端用于获取现场环境与供电状态；蜂鸣器告警端在超阈值或异常状态下给出声音提醒；供电端由多电源模块和稳压单元组成，为风速、风向等传感器以及主控和显示端提供稳定电能；显示端用于展示前端页面、航线规划结果、能耗预测结论和告警状态。

环境扰动与现场状态主要通过真实传感器采集和上位机任务数据注入两种方式进入系统。真实传感器包括温湿度气压一体传感器、风速风向传感器和电压传感器，用于形成温度、湿度、气压、风速、风向和供电状态的实时输入；PC 端可通过串口、USB 或网络接口向边缘端下发历史天气样本、候选航线和风险阈值，用于复现突发逆风、强侧风或低温等场景。显示告警单元由显示屏和蜂鸣器组成，分别用于展示预测能耗、剩余电量、风险等级以及超阈值声音提醒。

### 2.2.2 机械设计介绍

当前样机采用“固定式采集、显示与告警验证结构”，而不是完整无人机机体。选择该结构的原因有三点：首先，实物可以在室内安全演示传感器采集、页面显示与超阈值告警逻辑；其次，温湿度气压一体传感器、风速风向传感器、电压传感器和多电源模块布置清晰，便于展示 SC171V3 与外设之间的接线关系；最后，结构简单、布线直观，适合课程展示、现场讲解和离线验证。

机械结构从总体到局部可分为底座、传感安装区、控制告警区、供电区和显示区。底座负责承载和固定；传感安装区固定温湿度气压一体传感器、电压传感器以及风速风向传感器，保证采样位置稳定并便于走线；控制告警区固定 SC171V3 开发板和蜂鸣器；供电区布置多电源模块和稳压单元，其中多电源模块主要为风速、风向等传感器提供工作电源；显示区布置显示屏，用于展示前端页面和规划结果。该结构强调“传感器实时采集链路清晰”“显示屏可视化直观”和“蜂鸣器告警可演示”。

### 2.2.3 电路各模块介绍

电路部分按当前实物样机的真实接口关系划分为传感采集回路、蜂鸣器告警回路、显示回路和供电回路。供电回路由外部适配电源、多电源模块和 DC-DC 稳压单元组成，为 SC171V3、温湿度气压一体传感器、风速风向传感器、电压传感器、蜂鸣器和显示端提供稳定电能；传感采集回路把温度、湿度、气压、风速、风向和供电电压送入主控；告警回路由主控通过 GPIO 驱动蜂鸣器；显示回路通过显示屏呈现 Web 页面、候选航线和风险状态。

在信号连接上，温湿度气压一体传感器和风速风向传感器通过传感器接口或 RS485 总线与 SC171V3 通信，用于采集环境气象量；电压传感器通过电压采集接口向主控提供供电状态；蜂鸣器通过 GPIO 输出实现超阈值声音提醒；显示屏通过 HDMI、MIPI、USB 或网络页面方式显示前端结果；PC/云端通过 UART、USB 或网络接口下发天气剖面、候选航线和基准预测曲线。

关键输入信号包括系统供电输入 VBAT、5V/3.3V 稳压输出、RS485_A/B、VOLTAGE_SENSE、上位机任务参数 task_payload、天气剖面 weather_profile 和预测基准曲线 pred_curve。关键输出信号包括 GPIO_BUZZER、DISPLAY_OUT 和 log_upload。其中 GPIO_BUZZER 用于本地声音告警，DISPLAY_OUT 用于显示实时页面，log_upload 用于把采样数据和风险状态回传到上位机。由于当前未提供正式 SCH/PCB 文件，本文档先保留接口级框图位置，后续可替换为原理图、PCB 版图和实物连线照片。

## 2.3 软件系统介绍

软件系统采用“数据层—算法层—服务层—展示层”分层设计。数据层统一管理城市建筑、历史气象、板端测量值、仿真配置和航线结果；算法层完成低空气象重建、能耗预测、候选航线搜索和风险评估；服务层提供 HTTP、串口或文件交互接口；展示层由候选航线工作台、环境测量面板、虚拟验证控制台和板端显示页面组成。该设计使系统具备任务配置、结果生成、现场采集和前端复核的一体化能力。

# 第四部分 总结

## 4.1 可扩展之处

后续可从三方面扩展：一是接入更高精度的低空风场数据和真实飞行日志，使能耗模型从室内样机验证进一步过渡到外场校准；二是把 Web 工作台中的城市资产纠错、航线回放和误差曲线叠加做成稳定流程，形成可复现实验集；三是将单机规划扩展到多机协同任务，在电量约束、禁飞区、通信覆盖和气象风险之间进行联合调度。若条件允许，还可补充正式 CAD、SCH 和 PCB 文件，使机械和电路部分从接口级说明提升为可复刻的工程资料。

## 4.2 心得体会

本项目的难点不在于单独完成某一个算法页面或硬件连接，而在于把天气、城市、能耗、路径和嵌入式采集告警样机真正接成一个闭环。前期做模型时，容易只关注预测误差，但在城市低空任务中，误差往往来自多个环节：天气剖面是否合理、建筑高度是否可信、航线是否贴近障碍、风向是否与航向正确投影、板端传感器采样是否稳定，任何一个环节处理不好，最后都会表现为续航判断不可靠。因此我们在系统设计中把“可检查”作为重要目标，不只输出一个数值结论，而是尽量保留天气剖面、航线点列、风险摘要、仿真时序和 Web 场景，便于逐项排查。

在嵌入式联调中，SC171V3 的作用也逐渐明确。它不只是把 PC 端结果搬到板子上显示，而是承担本地状态判断、传感器采集、页面显示和蜂鸣器告警职责。温湿度气压一体传感器、风速风向传感器、电压传感器、显示刷新和告警输出之间需要统一节奏，否则界面上看似正确的预测曲线，未必能在样机上稳定反映出来。调试中我们尤其注意多电源模块供电稳定性、传感器接线可靠性、串口数据格式一致性以及 Web 工作台与板端状态同步。

Web 工作台的价值也在迭代中体现出来。只有把建筑、天气点和航线放到同一三维场景中，才能快速发现某些数据是否错位、采样高度是否不合理、航线是否穿越高楼密集区。它不是为了增加展示效果，而是帮助我们把验证对象变得可观察、可复核。后续工作将继续围绕实测数据校准、图纸补充和演示稳定性展开，使系统从“能运行”进一步提升到“能复现、能解释、能交付”。

# 无人机部署阶段预测增强模块

## 1. 定位

本文档只描述模型训练完成之后，在真实部署和实际飞行任务中可以加入的预测增强模块。

这些模块不属于当前模型训练主线，不用于解释数据集字段语义，也不应该混入当前训练实验的结论。它们的目标是在模型已经具备基础预测能力之后，通过真实设备、真实载荷、真实电池和真实环境的校准信息，降低单次任务的安全风险。

## 2. 与当前训练阶段的边界

当前训练阶段重点处理：

- 专家模型是否按飞行阶段拆分。
- 常量误差是否来自字段语义不统一。
- 无人机自身参数如何获得或推断。
- 数据集字段含义、单位、来源和可信度审计。
- 在输入字段正确的前提下，验证模型泛化能力和预测能力。

部署阶段增强模块重点处理：

- 某一台真实无人机、某一块电池、某一次载荷组合的实际偏差。
- 起飞前或任务早期的校准动作。
- 任务结束后的真实数据记录、回收和长期校准更新。
- 输出保守预测、误差区间和风险提示。

一句话区分：训练阶段要保证“输入正确时模型有泛化能力”；部署阶段要处理“真实设备当天状态和标称输入之间的偏差”。

## 3. 部署阶段增强模块

### 3.1 无人机配置档案

无人机配置档案用于描述一台具体无人机或一个机型的基础参数。

建议包含：

- 机型标识。
- 空机重量。
- 电池重量。
- 额定电池容量。
- 最大起飞重量。
- 旋翼数量。
- 电机和桨叶配置。
- 标称悬停功率。
- 标称巡航功率。
- 历史校准参数。

配置档案不替代训练数据字段审计。它只在部署阶段帮助系统把“当前真实飞机”映射到模型输入所需的统一工程参数。

### 3.2 起飞前原地悬停校准

原地悬停校准用于估计当前无人机、当前电池、当前载荷组合的基础功率偏移。

典型流程：

1. 起飞到固定安全高度。
2. 原地悬停 20 到 60 秒。
3. 记录电压、电流、功率、高度、垂向速度、姿态和风估计。
4. 计算实测悬停功率。
5. 与模型预测悬停功率比较。
6. 得到本次任务的功率偏移或缩放系数。

可能输出：

- `hover_power_offset_w`：悬停功率常量偏移。
- `hover_power_scale`：悬停功率缩放系数。
- `calibration_confidence`：校准可信度。

该模块适合修正重量、桨叶效率、电池状态、老化等导致的系统性偏差。

### 3.3 短程巡航校准

短程巡航校准用于估计水平飞行阶段的功率偏差。

典型流程：

1. 固定高度。
2. 固定航速。
3. 直线飞行 30 到 60 秒。
4. 记录实际功率、速度、航向、风估计和位置变化。
5. 与模型预测巡航功率比较。

可能输出：

- `cruise_power_offset_w`：巡航功率常量偏移。
- `cruise_power_scale`：巡航功率缩放系数。
- `wind_response_adjustment`：风响应修正量。

该模块不应该替代天气输入修正。它只用于处理当前飞机和当前环境下的实际巡航偏差。

### 3.4 任务结束数据记录和回收

每次任务结束后，系统应记录预测值和真实值的差异。

建议记录：

- 任务配置。
- 无人机配置。
- 电池配置。
- 载荷信息。
- 预测总电量。
- 实际总电量。
- 分阶段预测电量。
- 分阶段实际电量。
- 天气输入。
- 飞行日志。
- 校准参数。
- 风险判断结果。

这些数据用于后续长期校准和模型再训练，但不应该未经审计直接混入训练集。

### 3.5 任务级保守预测

部署阶段不应该只输出一个平均预测值。

建议输出：

- `P50`：中位预测，代表普通情况下的预测电量。
- `P90`：偏保守预测，代表较高消耗风险下的电量。
- `P95`：更保守预测，适合安全要求更高的任务。
- 最低建议剩余电量。
- 是否建议执行任务。
- 风险原因。

对于无人机任务，低估电量消耗比高估更危险。因此部署阶段必须显式处理保守预测和风险边界。

## 4. 与训练系统的接口

部署阶段增强模块不应该改变模型训练目标，但可以在推理时追加校准参数。

推荐接口：

输入：

- 统一后的无人机参数。
- 统一后的载荷参数。
- 路线和高度规划。
- 天气和风场输入。
- 起飞前或任务早期校准参数。
- 历史任务校准参数。

输出：

- 分段预测功率。
- 分段预测电量。
- 总预测电量。
- 保守电量预测。
- 剩余电量风险。
- 风险原因解释。

## 5. 不应混淆的事项

以下问题属于当前训练阶段，不属于本文档描述的部署增强模块：

- 某个数据集的重量字段到底是载荷还是总重。
- 某个数据集的高度字段到底是相对高度还是海拔高度。
- 风速风向是机载相对风还是环境风。
- 不同数据集电压、电流和功率计算口径是否一致。
- 专家模型是否应该按飞行阶段拆分。
- 常量误差是否来自输入字段语义错误。

这些问题必须在数据适配、字段语义审计和训练实验中解决。部署阶段校准只能修正真实设备的状态偏差，不能替代输入语义统一。

## 6. 当前建议

当前阶段不应该把部署增强模块混入训练实验。

训练阶段优先级：

- 完成重量、高度、风、功率字段语义审计。
- 判断常量误差是否来自输入字段错误。
- 尝试推断缺失的无人机自身参数。
- 将专家模型从数据集维度改为飞行阶段维度。
- 在输入正确的前提下验证模型泛化能力。

部署阶段后续优先级：

- 设计无人机配置档案。
- 设计原地悬停校准流程。
- 设计短程巡航校准流程。
- 设计任务结束数据记录和回收机制。
- 设计保守预测和风险输出。

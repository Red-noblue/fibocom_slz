# Q-learning 模块对外接口与独立前端设计

本文档说明 `modules/Q-learning` 从“策略实验代码”升级为“低空任务策略决策模块”时的第一版接口和前端边界。

## 当前定位

本模块负责把低空任务状态转成端-边-云卸载动作。它不替代无线地图、天气能耗、虚拟验证或正式三维前端，而是位于这些模块之上：

```text
无线地图 / 网络覆盖 -> link、覆盖概率、吞吐估计
天气能耗 -> battery、能耗约束、风险状态
虚拟验证 / 任务调度 -> queue、场景、仿真时序
任务语义 -> task_urgency、data_sensitivity、area_risk

上述状态 -> Q-learning 策略决策模块 -> local_tasks / edge_tasks / cloud_tasks
```

当前阶段只在本模块内部建设接口、文档和独立前端，不修改外部模块和共享环境。

## 对外状态字段

第一版接口使用离散状态，便于表格型 Q-learning、规则基线和未来 DQN/PPO 共用同一问题口径。

| 字段 | 含义 | 当前范围 | 典型来源 |
|---|---|---:|---|
| `queue` | 等待处理的任务队列长度 | `0..16` | 任务调度、虚拟验证 |
| `link` | 链路质量等级 | `0..2` | 无线地图、网络覆盖 |
| `battery` | 电量等级 | `0..4` | 天气能耗、仿真状态 |
| `edge_load` | 边缘节点负载等级 | `0..2` | MEC/基站侧状态 |
| `cloud_load` | 云端负载等级 | `0..2` | 云端状态或场景配置 |
| `task_urgency` | 任务紧急度 | `0..2` | 任务语义 |
| `data_sensitivity` | 数据敏感等级 | `0..2` | 任务语义、数据治理 |
| `area_risk` | 区域风险等级 | `0..2` | 低空风险评估、虚拟验证 |

## 对外动作字段

输出动作采用明确的三层命名：

| 字段 | 含义 |
|---|---|
| `local_tasks` | 本机处理任务数 |
| `edge_tasks` | 边缘侧处理任务数 |
| `cloud_tasks` | 云端处理任务数 |

内部历史代码中 `offload_tasks` 表示边缘卸载；对外接口统一用 `edge_tasks`，避免和云端卸载混淆。

## Python 接口

核心入口位于：

```text
src/qlearning_policy/interface.py
```

主要对象：

- `PolicyStatePayload`：模块间输入状态。
- `PolicyActionPayload`：对外动作。
- `PolicyDecisionPayload`：完整决策结果，包含动作类别、估算 reward、delay、energy 和解释提示。
- `build_interface_contract()`：输出接口契约。
- `decide_policy()`：输入状态，输出一次策略决策。

无 `policy_table.csv` 时，`decide_policy()` 使用可解释规则基线；传入训练后的 `policy_table.csv` 时，按精确状态查表。

## CLI 接口

查看接口契约：

```bash
cd modules/Q-learning
PYTHONPATH=src ../../._envs/uav-modules-py310/bin/python -m qlearning_policy describe-interface
```

执行一次规则基线决策：

```bash
PYTHONPATH=src ../../._envs/uav-modules-py310/bin/python -m qlearning_policy decide \
  --coverage-scenario congested_edge \
  --queue 10 \
  --link 2 \
  --battery 4 \
  --edge-load 2 \
  --cloud-load 0 \
  --task-urgency 1 \
  --data-sensitivity 0 \
  --area-risk 0
```

使用训练后的策略表：

```bash
PYTHONPATH=src ../../._envs/uav-modules-py310/bin/python -m qlearning_policy decide \
  --policy-table outputs/qlearning_policy/scenario_sweep/balanced/policy_table.csv \
  --queue 8 \
  --link 1 \
  --battery 4 \
  --edge-load 1 \
  --cloud-load 1 \
  --task-urgency 1 \
  --data-sensitivity 1 \
  --area-risk 1
```

导出前端仪表盘数据：

```bash
PYTHONPATH=src ../../._envs/uav-modules-py310/bin/python -m qlearning_policy export-dashboard-data
```

默认输出：

```text
web/static/data/policy-dashboard.json
```

## 独立前端

前端目录：

```text
web/
```

该前端只展示 Q-learning 模块的策略实验、接口字段和轻量试算，不接入 `uav_virtual_validation` 的三维控制台，也不修改其他模块。

前端当前包含：

- 方案 3 三层卸载稳定性结果展示。
- 对外状态字段和动作字段说明。
- 多场景 Q-learning 指标条形展示。
- 本地规则试算器，用于理解接口字段如何影响动作。

启动方式：

```bash
cd modules/Q-learning
PYTHONPATH=src ../../._envs/uav-modules-py310/bin/python -m qlearning_policy export-dashboard-data
python3 -m http.server 8091 --directory web
```

访问：

```text
http://127.0.0.1:8091/
```

## 后续升级建议

1. 先固定当前接口契约，不急于引入 HTTP 服务。
2. 后续由无线地图模块输出 `link` 或连续链路指标，再在本模块内做离散化适配。
3. 由天气能耗模块输出 `battery`、能耗余量和风险，再映射到当前状态字段。
4. 等外部模块产物稳定后，再考虑把 `decide_policy()` 包装成 HTTP API 或仿真回调。
5. 前端继续保持模块独立，只消费本模块导出的 JSON，不直接读取其他模块内部文件。

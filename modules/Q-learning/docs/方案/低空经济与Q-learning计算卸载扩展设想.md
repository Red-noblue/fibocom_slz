# 面向低空经济的 Q-learning 计算卸载扩展设想

本文用于整理“基于 Q-learning 的计算卸载策略”作业的扩展方向：把原始移动边缘计算卸载模型，迁移到低空经济中的无人机端边协同计算场景。核心问题是：当无人机同时面对通信质量变化、任务随机到达、电量限制、本地算力限制和边缘节点负载变化时，如何决定“本地计算、卸载到基站边缘节点、或采用混合计算”。

## 1. 核心判断

低空经济中确实存在与本作业高度相关的问题：无人机终端算力、电量和载荷有限，而巡检、物流、安防、测绘、应急等任务又会产生大量视频、图像、感知和路径规划计算需求。因此，“计算是否卸载、卸载到哪里、何时切回本地”是一个真实存在的调度问题。

但是需要区分两件事：

- `边缘计算、5G-A、通感一体化、低空智联网` 已经是产业和政策讨论中的重要方向。
- `Q-learning/强化学习` 在无人机边缘计算卸载论文中很常见，但公开产业系统通常不会直接宣称“使用 Q-learning 调度无人机任务”，真实商用落地会更强调稳定性、安全边界、可解释规则和工程冗余。

因此，更合理的表述是：

```text
低空经济中的无人机计算卸载问题与 Q-learning 建模高度契合。
本作业可以把 Q-learning 作为一种在线策略学习方法，
用于探索无人机在端边云协同场景下的动态计算决策。
```

## 2. 为什么 5G 发达后，无人机仍然不能完全不要算力

一个容易出现的误解是：如果未来 5G/5G-A/6G 通信足够发达，无人机是不是可以不需要本地算力，把所有计算都交给基站或云端。

结论是否定的。更现实的未来形态是：

```text
无人机本地算力 + 基站边缘算力 + 区域云/中心云
共同协同
```

无人机必须保留本地算力，原因是安全关键任务不能完全依赖网络：

- 飞控稳定、姿态控制、返航和降落必须本地闭环完成。
- 避障、紧急制动、近距离安全感知需要毫秒级响应。
- 通信中断、遮挡、干扰、基站拥塞时，必须有降级运行能力。
- 部分数据涉及隐私、监管或安全要求，不能无条件上传。

边缘节点适合处理高算力但可容忍一定通信链路的任务：

- 高清视频分析和目标识别。
- 多无人机协同路径规划。
- 区域低空态势融合。
- 城市级航路调度。
- 大范围地图匹配、数字孪生和监管数据融合。
- 非紧急任务的批量处理。

所以真正的问题不是“本地还是云端”，而是：

```text
当前这一刻，这个任务，应该在哪里计算？
```

这正是 Q-learning 可以建模的问题。

## 3. 与低空经济相关的现实问题

从中国低空经济的发展方向看，和本作业最相关的问题主要有六类。

### 3.1 低空通信覆盖和链路稳定性

低空飞行高度、建筑遮挡、城市峡谷、多径传播和高速移动都会影响通信质量。即使 5G-A 覆盖增强，也不能假设所有区域、所有高度、所有时刻都有稳定高速链路。

对计算卸载的影响是：

- 信道好时，卸载大任务可能更划算。
- 信道差时，上传数据可能带来更大时延和能耗。
- 链路波动时，固定“总是卸载”策略会不稳定。

### 3.2 边缘节点负载波动

低空经济不是单架无人机问题。未来同一区域可能同时存在物流无人机、巡检无人机、警用无人机、测绘无人机和低空载人飞行器。即使基站边缘节点有算力，也会出现排队、负载高峰和资源竞争。

对计算卸载的影响是：

- 边缘负载低时，卸载收益更高。
- 边缘负载高时，本地计算可能更稳定。
- 策略需要同时考虑通信质量和边缘算力状态。

### 3.3 电量与任务载荷限制

无人机电量有限，计算、通信、飞行都会耗电。卸载不一定省电，因为上传高清视频或传感数据本身也要消耗发射功率。

对计算卸载的影响是：

- 本地计算消耗芯片能量。
- 卸载计算消耗通信能量。
- 电量低时，策略可能要优先保留返航和安全余量。

### 3.4 安全监管与实时性约束

低空飞行涉及公共安全，不能只追求平均 reward。某些任务一旦超时，后果比普通任务更严重。

例如：

- 避障任务超时会直接带来碰撞风险。
- 巡检图像识别稍慢可能只是降低效率。
- 监管告警任务需要高可靠和可追踪。

因此，任务类型和 deadline 应该成为扩展状态或 reward 的一部分。

### 3.5 数据安全与隐私

低空设备可能采集城市道路、园区、工厂、人员和车辆数据。全部上传到云端并不总是合适。

对计算卸载的影响是：

- 敏感数据可以优先本地预处理或脱敏。
- 只上传特征、告警结果或压缩摘要。
- reward 中可以加入数据安全惩罚项。

### 3.6 异构设备和通感算一体化

未来低空网络可能同时具备通信、感知、定位和计算能力。基站不只是“转发数据”，还可能参与感知目标、跟踪轨迹、融合多源信息、调度边缘算力。

这对应“通感算一体化”的工程方向：

```text
通信：提供数据传输链路
感知：识别低空目标、位置、轨迹和环境
计算：在边缘节点处理任务、融合态势、辅助决策
```

Q-learning 可以作为其中一个决策层方法，用来学习端侧和边侧计算资源之间的切换策略。

## 4. 如何把原始 Q-learning 作业迁移到低空无人机场景

原始作业中的建模大致是：

```text
状态 state = (队列长度 q, 传输速率 r)
动作 action = (本地处理任务数, 卸载任务数)
奖励 reward = 任务收益 - 能耗惩罚 - 时延惩罚
```

迁移到低空经济无人机场景后，可以扩展为：

```text
状态 state =
(任务队列 q,
 信道质量 r 或 SNR,
 无人机电量 battery,
 本地算力 f_local,
 边缘节点负载 edge_load,
 任务紧急度 deadline,
 飞行区域 area,
 安全风险 risk)
```

动作可以扩展为：

```text
action =
本地计算
卸载到最近基站
卸载到区域边缘节点
上传中心云
部分本地 + 部分卸载
安全关键任务强制本地计算
```

奖励函数可以扩展为：

```text
reward =
任务完成收益
- 时延惩罚
- 本地计算能耗
- 通信上传能耗
- 边缘排队惩罚
- 超时惩罚
- 安全风险惩罚
- 数据敏感性惩罚
```

状态转移可以设计为：

```text
下一时刻队列长度 = 当前队列 + 新到达任务 - 已完成任务
下一时刻信道质量 = 根据区域和随机扰动变化
下一时刻电量 = 当前电量 - 飞行能耗 - 计算能耗 - 通信能耗
下一时刻边缘负载 = 当前负载 + 其他无人机任务扰动 - 边缘完成任务
```

这样，原始课程作业就从普通移动终端计算卸载，升级为：

```text
面向低空经济无人机的通感算协同计算卸载策略
```

## 5. 为什么 Q-learning 合适

Q-learning 适合这个问题，是因为它不要求提前知道完整环境模型。无人机实际运行时，任务到达、通信质量、边缘负载和电量变化都具有随机性，很难用固定公式一次性写出最优策略。

Q-learning 学习的是一个表：

```text
Q(state, action)
```

含义是：

```text
在某个状态下采取某个动作，从长期看有多值得。
```

更新公式可以写成：

```text
Q(s, a) <- Q(s, a) + alpha * [reward + gamma * max Q(s_next, a_next) - Q(s, a)]
```

浅显理解：

- `reward` 是这一步马上拿到的收益。
- `max Q(s_next, a_next)` 是下一步还能带来的最好未来收益。
- `alpha` 控制学习速度。
- `gamma` 控制是否重视长期收益。

所以它不是只看眼前，而是在学习：

```text
现在这么决策，未来会不会更好？
```

在无人机场景中，它可以学出类似规律：

- 网络好、边缘负载低、任务不紧急时，倾向于卸载。
- 网络差、任务紧急、安全风险高时，倾向于本地计算。
- 电量低时，避免高能耗动作，优先保障返航和安全任务。
- 队列堆积严重时，策略可能更积极地处理任务，避免时延爆炸。

## 6. 可以设计的实验对比

为了让报告更有说服力，可以比较以下策略。

`Local Only`

全部任务本地计算。优点是稳定、不依赖网络；缺点是本地算力和能耗压力大。

`Always Offload`

只要有任务就尽量卸载。优点是利用边缘算力；缺点是对通信质量和边缘负载过于敏感。

`Rule-Based`

用固定规则决策。例如 SNR 高于阈值就卸载，否则本地计算。优点是可解释；缺点是阈值僵硬。

`Q-learning Adaptive`

根据队列、信道、电量、边缘负载等状态动态选择动作。目标是获得更高长期平均收益。

预期结果可以写成：

```text
在网络稳定且边缘负载低时，Always Offload 可能表现较好。
在网络差或边缘拥塞时，Local Only 可能更稳定。
在信道、负载和任务到达都随机变化时，Q-learning Adaptive 更有机会取得综合最优。
```

## 7. 可落地的作业扩展方案

如果当前作业已经完成基础版本，可以按下面顺序扩展。

### 7.1 低成本扩展：只改状态

在原始状态 `(q, r)` 基础上加入本地算力档位：

```text
state = (q, r, f_level)
```

解释：

- `f_level` 高时，本地计算更划算。
- `f_level` 低时，卸载可能更划算。
- 这对应无人机芯片算力、温度降频、节能模式等实际情况。

### 7.2 中等扩展：加入边缘负载

加入边缘节点负载：

```text
state = (q, r, edge_load)
```

解释：

- 边缘负载低时，卸载任务处理快。
- 边缘负载高时，卸载会排队。
- 这比只看无线速率更接近真实低空网络。

### 7.3 高价值扩展：加入任务紧急度

加入任务 deadline 或任务类型：

```text
state = (q, r, task_urgency)
```

reward 中加入超时惩罚：

```text
reward = 原始收益 - lambda_deadline * 超时任务数
```

解释：

- 避障、告警、飞控辅助属于高紧急任务。
- 巡检图像归档、日志上传属于低紧急任务。
- 策略应该对不同任务做不同选择。

### 7.4 冲高分扩展：端边云三层动作

把动作从“本地/卸载”扩展成三层：

```text
action in {local, edge, cloud}
```

解释：

- `local` 低时延，但本地能耗高、算力有限。
- `edge` 时延适中，适合实时感知和区域协同。
- `cloud` 算力强，但网络链路长、时延高，适合非实时任务。

这会让报告主题更贴近“通感算一体化”和“端边云协同”。

## 8. 推荐报告题目与表述

可选题目：

```text
面向低空经济无人机的 Q-learning 端边协同计算卸载策略研究
```

或者：

```text
通感算一体化场景下基于 Q-learning 的无人机计算卸载策略扩展
```

报告中可以使用的核心表述：

```text
未来低空经济不会走向无人机完全无算力化，而会走向端边云协同。
无人机端负责安全关键和实时闭环任务，基站边缘节点负责高算力感知、路径规划、态势融合和多机协同。
由于低空通信质量、边缘负载、任务到达和电量状态具有随机性，固定规则策略难以长期最优。
因此，可以将卸载决策建模为马尔可夫决策过程，并使用 Q-learning 学习在线卸载策略。
```

## 9. 技术边界

写报告时需要注意边界，否则容易显得过度夸张。

- 表格型 Q-learning 适合状态空间较小的课程实验。
- 真实低空经济系统的状态空间会更大，可能需要 DQN、多智能体强化学习或模型预测控制。
- 安全关键任务不能完全交给黑盒强化学习，需要规则约束、安全屏障和人工监管机制。
- NPU 更适合神经网络推理，不是表格型 Q-learning 的刚需。
- GPU/XPU 对批量仿真、深度强化学习和大规模参数扫描更有价值。

因此，本作业中最稳妥的定位是：

```text
用 Q-learning 验证低空无人机计算卸载的在线决策思想，
而不是宣称它已经可以直接替代真实低空监管和飞控系统。
```

## 10. 后续可做工作

后续如果继续推进，可以按以下路线落地：

1. 把原始 `qlearning.m` 改写为 Python 工程。
2. 先复现课程原始 `(q, r)` 状态和本地/卸载动作。
3. 增加 `edge_load` 或 `battery` 状态，形成低空经济扩展版本。
4. 对比 `Local Only`、`Always Offload`、`Rule-Based`、`Q-learning Adaptive`。
5. 输出平均 reward、平均时延、平均能耗、队列长度、任务完成率等指标。
6. 在报告中解释不同策略在不同通信质量和边缘负载下的行为差异。

最推荐的最小扩展版本是：

```text
state = (q, r, edge_load)
action = (local_tasks, offload_tasks)
reward = 任务收益 - 本地能耗 - 传输能耗 - 边缘排队时延 - 队列积压惩罚
```

这个版本改动不算大，但能自然连接低空经济、5G-A、通感算一体化和边缘计算卸载，适合作为课程作业的高分扩展方向。

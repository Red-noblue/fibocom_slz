如果只想快速查看所有路线的可复制命令，优先阅读：

```text
../各路线运行示例.md
```

## SNPE GPU INT8 主线

更完整的运行示例见 `../各路线运行示例.md`。

单样本复现：

```bash
._envs/radio-map-qcs6490-py38/bin/python \
  modules/radio_map_qcs6490_deploy/scripts/run_snpe_gpu_inference.py \
  --model modules/fiboaistack_229_env/outputs/snpe_quant_qemu82_full_20260703_075537/radio_map_liteunet_quantized.dlc \
  --features-dir modules/radio-map-estimation-workbench/modules/m2/runs/datas/features/v1_base_256_k1_landuse3 \
  --sample-id 166_k0 \
  --runtime GPU \
  --mode 5 \
  --warmup 1 \
  --repeat 3 \
  --input-name input \
  --output-name output \
  --output-dir modules/radio_map_qcs6490_deploy/outputs/manual_snpe_gpu_int8
```

使用 QAIRT INT8 产物：

```bash

  modules/radio_map_qcs6490_deploy/scripts/run_snpe_gpu_inference.py \
  --model modules/fiboaistack_229_env/outputs/qairt_quant_qemu82_full_20260703_080039/radio_map_liteunet_qairt_quantized.dlc \
  --features-dir modules/radio-map-estimation-workbench/modules/m2/runs/datas/features/v1_base_256_k1_landuse3 \
  --sample-id 166_k0 \
  --runtime GPU \
  --mode 5 \
  --warmup 1 \
  --repeat 3 \
  --output-dir modules/radio_map_qcs6490_deploy/outputs/manual_qairt_gpu_int8
```

## QNN DSP Pad+INT8 专项路线

这条路线已经能执行完整 radio-map 模型，但当前精度弱于 SNPE DSP INT8，因此只作为 HTP/CDSP 专项研究线。

更完整的运行示例见 `../各路线运行示例.md`。

```bash
._envs/radio-map-qcs6490-py38/bin/python \
  modules/radio_map_qcs6490_deploy/scripts/run_qnn_radio_map_inference.py \
  --model modules/fiboaistack_229_env/outputs/qnn_htp_rewrite_pad_constant_int8/host_build/libs/aarch64-ubuntu-gcc9.4/libradio_map_liteunet_pad_constant_int8_qnn.so \
  --qnn-json modules/fiboaistack_229_env/outputs/qnn_htp_rewrite_pad_constant_int8/radio_map_liteunet_pad_constant_int8_qnn_net.json \
  --features-dir modules/radio-map-estimation-workbench/modules/m2/runs/datas/features/v1_base_256_k1_landuse3 \
  --sample-id 166_k0 \
  --runtime DSP \
  --warmup 1 \
  --repeat 3 \
  --log-level ERROR \
  --output-dir modules/radio_map_qcs6490_deploy/outputs/manual_qnn_pad_constant_int8_dsp
```

注意：QNN INT8 `.so` 需要按 QNN JSON 中的 input/output `scale` 和 `offset` 走 uint8 量化/反量化路径，不能继续使用 float 输入输出接口。

## 批量复现实验命令

当前已固化专用批量入口：

```bash
._envs/radio-map-qcs6490-py38/bin/python \
  modules/radio_map_qcs6490_deploy/scripts/batch_snpe_int8_repro.py \
  --limit 50 \
  --warmup 1 \
  --repeat 1 \
  --output-dir modules/radio_map_qcs6490_deploy/outputs/int8_repro_test50_$(date +%Y%m%d_%H%M%S)
```

默认会运行三条链路：

```text
fp32_gpu,int8_gpu,int8_dsp
```

如果只想复测默认主线和速度副主线：

```bash
._envs/radio-map-qcs6490-py38/bin/python \
  modules/radio_map_qcs6490_deploy/scripts/batch_snpe_int8_repro.py \
  --limit 50 \
  --cases int8_gpu,int8_dsp \
  --warmup 1 \
  --repeat 1 \
  --output-dir modules/radio_map_qcs6490_deploy/outputs/int8_gpu_dsp_repro_test50_$(date +%Y%m%d_%H%M%S)
```

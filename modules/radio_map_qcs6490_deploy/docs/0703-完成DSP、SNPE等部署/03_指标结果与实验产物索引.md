
样本数：`50` 个 test 样本，成功 `50/50`。

| 路线 | 平均耗时 | RMSE | MAE | 相对 FP32 GPU 输出 RMSE | 相对 INT8 GPU 加速 |
| --- | ---: | ---: | ---: | ---: | ---: |

判断：50 样本结果支持 20 样本结论。SNPE GPU INT8 相对 FP32 GPU 的输出差异仍很小；SNPE DSP INT8 速度优势稳定，但精度损失仍明显高于 GPU INT8。

## 100 样本 INT8 GPU/DSP 固化复现实验

实验目录：

```text
modules/radio_map_qcs6490_deploy/outputs/int8_gpu_dsp_repro_test100_20260703_133812
```

运行命令：

```bash
python3 modules/radio_map_qcs6490_deploy/scripts/batch_snpe_int8_repro.py \
  --limit 100 \
  --cases int8_gpu,int8_dsp \
  --warmup 1 \
  --repeat 1 \
  --output-dir modules/radio_map_qcs6490_deploy/outputs/int8_gpu_dsp_repro_test100_20260703_133812
```

样本数：`100` 个 test 样本，成功 `100/100`。本轮没有重跑 FP32 GPU 配对输出，因此只统计 INT8 GPU/DSP 标签误差和 DSP 相对 INT8 GPU 的输出差异。

| 路线 | 平均耗时 | RMSE | MAE | 相对 INT8 GPU 输出 RMSE | 相对 INT8 GPU 加速 |
| --- | ---: | ---: | ---: | ---: | ---: |
| SNPE GPU INT8 | `312.34 ms` | `3.475 dB` | `2.185 dB` | 基线 | - |
| SNPE DSP INT8 | `86.78 ms` | `3.834 dB` | `2.399 dB` | `1.715 dB` | `3.62x` |

判断：100 样本结果继续支持当前路线分层。SNPE GPU INT8 是默认精度模式；当前认为 SNPE DSP INT8 相对 GPU 约 `1.715 dB RMSE` 的输出差异可以接受，因此 SNPE DSP INT8 可作为高性能模式使用。

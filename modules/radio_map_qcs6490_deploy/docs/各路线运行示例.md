# 各路线运行示例

本文集中记录当前 SC171V3 / QCS6490 设备上 radio-map 模型各条部署路线的运行命令。命令默认从仓库根目录 `/home/fibo/fibocom_slz` 执行。

## 运行前检查

先确认本机 SDK、模型产物、样本目录和后端判据是否齐全：

```bash
._envs/radio-map-qcs6490-py38/bin/python \
  modules/radio_map_qcs6490_deploy/scripts/check_deploy_readiness.py
```

常用公共参数：

| 参数 | 当前值 |
| --- | --- |
| Python 环境 | `._envs/radio-map-qcs6490-py38/bin/python` |
| 输入张量 | `input` |
| 输出张量 | `output` |
| 输入 shape | `1,256,256,7`，NHWC |
| Fibo AI Stack mode | `5` |
| 默认真实样本目录 | `modules/radio-map-estimation-workbench/modules/m2/runs/datas/features/v1_base_256_k1_landuse3` |

## 路线总览

| 路线 | 当前定位 | 推荐场景 |
| --- | --- | --- |
| SNPE GPU INT8 DLC | 默认主线 | 日常本机部署、精度优先、复现实验。 |
| SNPE DSP INT8 DLC | 速度副主线 | 极端速度优先，并能接受相对 GPU 约 `1.7 dB RMSE` 输出差异。 |
| SNPE GPU FP32 DLC | 精度基线 | 对比 INT8 精度损失或排查量化问题。 |
| QAIRT GPU/DSP INT8 DLC | 替代量化产物 | 对照 SNPE 量化 DLC。 |
| QNN GPU `.so` | 备用验证线 | 验证 QNN `.so` 完整模型是否可运行。 |
| QNN DSP Pad+INT8 `.so` | HTP/CDSP 专项研究线 | 研究 QNN DSP 图改写和精度损失，不作为正式主线。 |
| UAV route profile | 应用输出示例 | 将航线输入批量转成 `route_radio_profile.json`。 |

## SNPE GPU INT8 默认主线

单样本真实 M2 输入：

```bash
._envs/radio-map-qcs6490-py38/bin/python \
  modules/radio_map_qcs6490_deploy/scripts/run_snpe_gpu_inference.py \
  --model modules/fiboaistack_229_env/outputs/snpe_quant_qemu82_full_20260703_075537/radio_map_liteunet_quantized.dlc \
  --features-dir modules/radio-map-estimation-workbench/modules/m2/runs/datas/features/v1_base_256_k1_landuse3 \
  --sample-id 166_k0 \
  --runtime GPU \
  --mode 5 \
  --warmup 1 \
  --repeat 3 \
  --input-name input \
  --output-name output \
  --output-dir modules/radio_map_qcs6490_deploy/outputs/manual_snpe_gpu_int8_$(date +%Y%m%d_%H%M%S)
```

输出：

- `<sample_id>_gpu_snpe.npy`
- `<sample_id>_gpu_snpe_summary.json`

## SNPE DSP INT8 速度副主线

单样本真实 M2 输入：

```bash
._envs/radio-map-qcs6490-py38/bin/python \
  modules/radio_map_qcs6490_deploy/scripts/run_snpe_gpu_inference.py \
  --model modules/fiboaistack_229_env/outputs/snpe_quant_qemu82_full_20260703_075537/radio_map_liteunet_quantized.dlc \
  --features-dir modules/radio-map-estimation-workbench/modules/m2/runs/datas/features/v1_base_256_k1_landuse3 \
  --sample-id 166_k0 \
  --runtime DSP \
  --mode 5 \
  --warmup 1 \
  --repeat 3 \
  --input-name input \
  --output-name output \
  --output-dir modules/radio_map_qcs6490_deploy/outputs/manual_snpe_dsp_int8_$(date +%Y%m%d_%H%M%S)
```

使用口径：

- 这是当前高性能模式，100 样本平均约 `86.78 ms`。
- 使用时应同时披露相对 SNPE GPU INT8 约 `1.715 dB RMSE` 的输出差异。
- 不要把 SNPE HTP/NPU 的初始化成功和 SNPE DSP INT8 的真实 CDSP/FastRPC 证据混为一谈。

## SNPE DSP INT8 强证据复核

需要确认是否真的进入 CDSP/FastRPC 时运行：

```bash
strace -f \
  -o modules/radio_map_qcs6490_deploy/outputs/manual_snpe_dsp_int8_$(date +%Y%m%d_%H%M%S).trace \
  -s 256 \
  -e trace=openat,access,newfstatat,ioctl,connect \
  ._envs/radio-map-qcs6490-py38/bin/python \
  modules/radio_map_qcs6490_deploy/scripts/probe_fibo_dlc_runtime.py \
  --model modules/fiboaistack_229_env/outputs/snpe_quant_qemu82_full_20260703_075537/radio_map_liteunet_quantized.dlc \
  --framework snpe \
  --runtime DSP \
  --input-shape 1,256,256,7 \
  --input-name input \
  --output-name output \
  --mode 5 \
  --warmup 1 \
  --repeat 5 \
  --sample-device \
  --log-level TRACE
```

重点检查 trace 中是否出现：

```text
/dev/adsprpc-smd-secure
/dsp/cdsp
libSnpeHtpV68Stub.so
libSnpeHtpV68Skel.so
FASTRPC_IOCTL_ALLOC_DMA_BUFF
```

## QNN GPU `.so` 备用验证线

QNN 路线不能加载 DLC，必须使用 `.so` 模型库：

```bash
._envs/radio-map-qcs6490-py38/bin/python \
  modules/radio_map_qcs6490_deploy/scripts/probe_fibo_dlc_runtime.py \
  --model modules/fiboaistack_229_env/outputs/qnn_model/lib/aarch64-ubuntu-gcc9.4/libradio_map_liteunet_qnn.so \
  --framework qnn \
  --runtime GPU \
  --input-shape 1,256,256,7 \
  --input-name input \
  --output-name output \
  --mode 5 \
  --warmup 1 \
  --repeat 3 \
  --sample-device \
  --log-level TRACE
```

用途：

- 验证 QNN `.so` 完整模型可运行。
- 作为 QNN DSP 专项的 GPU 对照，不替代 SNPE GPU INT8 主线。

## QNN DSP Pad+INT8 专项路线

这条路线已能执行完整模型，但精度弱于 SNPE DSP INT8，当前只作为 HTP/CDSP 专项研究线：

```bash
._envs/radio-map-qcs6490-py38/bin/python \
  modules/radio_map_qcs6490_deploy/scripts/run_qnn_radio_map_inference.py \
  --model modules/fiboaistack_229_env/outputs/qnn_htp_rewrite_pad_constant_int8/host_build/libs/aarch64-ubuntu-gcc9.4/libradio_map_liteunet_pad_constant_int8_qnn.so \
  --qnn-json modules/fiboaistack_229_env/outputs/qnn_htp_rewrite_pad_constant_int8/radio_map_liteunet_pad_constant_int8_qnn_net.json \
  --features-dir modules/radio-map-estimation-workbench/modules/m2/runs/datas/features/v1_base_256_k1_landuse3 \
  --sample-id 166_k0 \
  --runtime DSP \
  --warmup 1 \
  --repeat 3 \
  --log-level ERROR \
  --output-dir modules/radio_map_qcs6490_deploy/outputs/manual_qnn_pad_constant_int8_dsp_$(date +%Y%m%d_%H%M%S)
```

注意：

- QNN INT8 `.so` 必须使用 QNN JSON 中的 input/output `scale` 和 `offset` 做 uint8 量化/反量化。
- 不要用未改写 QNN `.so` 反复测试 DSP；未改写路线已知会卡在 Pad / op validation。

## CPU Fallback 判据检查

不要只看 `Init=0` 或 `Execute=0`。对后端是否真实加速，应结合日志、耗时、输出差异和设备侧证据：

```bash
._envs/radio-map-qcs6490-py38/bin/python \
  modules/radio_map_qcs6490_deploy/scripts/judge_backend_fallback.py \
  --run-dir modules/radio_map_qcs6490_deploy/outputs/snpe_backend_judgement_20260703_044730 \
  --runtime-key GPU
```

当前结论：

- SNPE GPU INT8：真实 Adreno GPU 路线。
- SNPE DSP INT8：有 CDSP/FastRPC 证据，作为速度副主线。
- SNPE HTP/NPU：仍按疑似 CPU fallback 处理。
- QNN DSP Pad+INT8：可执行但精度较弱，作为专项研究线。

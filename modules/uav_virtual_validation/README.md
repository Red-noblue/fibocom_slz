# UAV 虚拟验证项目

本子项目用于在当前工作空间内建设无人机虚拟世界、仿真执行与模型验证链路。

项目目标不是实现飞控系统，而是为天气驱动能耗预测模型提供可量化、可复现、可逐步升级的虚拟验证环境。

## 当前边界

- `temp/fibo` 是历史开发板嵌入式系统端项目，只作为参考，不作为当前主开发目录。
- 本项目先实现本机可运行的简化仿真闭环。
- 后续再逐步接入 PX4/Gazebo、AirSim、Isaac Sim 或真实日志校准。

## 目录说明

```text
configs/   仿真场景、飞行器、环境、天气配置
data/      原始数据、外部数据、中间数据、处理后数据
docs/      中文文档入口，按总览、操作、数据、验证、规划分类
src/       Python 包源码
scripts/   可执行脚本入口
outputs/   仿真与验证输出产物
tests/     测试
```

## 城市要素纠错版本

Web 控制台右侧的“城市要素纠错”用于人工修正被误识别为建筑的城市对象，不做建筑用途分类。纠错结果保存在 `outputs/real_city_corrections/`，页面按当前城市过滤版本列表。点击“保存为新版本”或“覆盖当前版本”后，会同步生成对应城市的 `outputs/real_city/<城市>/corrected_tiles/<版本>/`，让低矮地面要素对应的原错误建筑从正式加载资产中消失。原始版本禁止覆盖和删除；非原始版本可覆盖、另存或删除，删除当前版本需要连续点击两次确认。详细规则见 `docs/操作/城市纠错.md`。

## 阶段目标

1. 简化仿真器：生成虚拟真值时序数据。
2. 预测对比：计算预测结果与仿真结果误差。
3. 展示适配：输出可供 GIS/回放展示消费的统一产物。
4. 高保真升级：预留 PX4/Gazebo、AirSim、Isaac Sim 接口。
5. 真实校准：使用真实飞行日志或台架实验数据缩小 sim-to-real gap。

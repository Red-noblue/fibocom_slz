# 项目定义与答辩边界

## 1. 项目一句话定义

本项目是一个面向城市低空无人机任务的天气驱动能耗预测与虚拟验证系统。

它不做无人机飞控，也不声称虚拟世界天然等同真实世界，而是解决一个更明确的问题：

```text
在任务起飞前，基于航线、载荷、电池参数和天气预报，预测本次任务的能耗、剩余电量和风险边界，并通过虚拟仿真与后续真实数据校准验证预测可靠性。
```

## 2. 本项目不是什么

本项目不是：

- 飞控系统
- 无人机动力学仿真器
- 单纯 3D 展示系统
- 用虚拟数据自证正确的闭环造假系统
- 替代真实试飞的绝对真值系统

这些边界必须讲清楚。否则答辩时很容易被问到飞控、姿态、电机、仿真真实性，然后整个项目被带偏。

## 3. 本项目是什么

本项目包括三层：

```text
预测层：任务级天气能耗预测模型
验证层：虚拟仿真、误差计算、场景消融
展示层：路线、曲线、风险和 3D 环境可视化
```

其中核心创新不在“造一个无人机游戏世界”，而在：

- 把天气因素引入起飞前任务级能耗预测
- 给出剩余电量和风险边界
- 支持不同天气、航线、高度、载荷条件下的仿真验证
- 后续可通过真实日志或台架实验持续校准

## 4. 预测模型和仿真模型的区别

答辩中最容易被攻击的问题是：

```text
既然虚拟世界也要生成电量数据，那你们的预测模型是不是重复了？
```

标准回答：

```text
不重复。仿真模型是虚拟实验环境，用来生成更细粒度的“被测对象响应”；预测模型是任务规划阶段的轻量模型，用来在起飞前根据可获得信息估计能耗和风险。
二者输入边界、使用阶段和复杂度不同。
```

更具体地说：

| 对比项 | 虚拟仿真真值模型 | 任务级预测模型 |
|---|---|---|
| 角色 | 虚拟实验场 / 考官 | 待验证算法 / 考生 |
| 使用阶段 | 验证阶段 | 起飞前规划阶段 |
| 输入信息 | 细粒度状态、扰动、局地风、风险区、随机变化 | 航线、计划速度、高度、载荷、电池、天气预报 |
| 输出 | 时序电流、电压、功率、电量、风险状态 | 总能耗、剩余电量、风险评分 |
| 复杂度 | 可以复杂，不部署到机端 | 应轻量、可解释、可部署 |
| 目的 | 产生可对比观测数据 | 做任务可达性评估 |

## 5. 为什么虚拟验证仍然有价值

由于当前本地禁飞，无法直接进行大量真实外场试飞。

因此项目采用分阶段验证路线：

```text
简化仿真验证
  -> 高保真仿真验证
  -> 台架/小规模实验校准
  -> 真实飞行日志校准
```

虚拟验证的价值不是证明“绝对真实”，而是：

- 快速覆盖多种天气与任务场景
- 验证模型对风、温度、载荷、高度变化的趋势响应
- 提前发现风险判定和剩余电量估计问题
- 为后续真实数据校准提供统一指标和接口

## 6. 答辩中不要说什么

不要说：

```text
我们的仿真和现实误差极低。
虚拟世界可以完全替代真实飞行。
仿真数据证明模型一定准确。
```

这些说法太容易被反问：

- 误差是多少？
- 和哪台无人机对比？
- 哪些真实日志验证过？
- 风场怎么建模？
- 电机和电池参数怎么标定？

## 7. 答辩中应该怎么说

推荐表述：

```text
在当前禁飞条件下，我们没有把虚拟仿真当成真实飞行的替代品，而是把它作为可控实验环境。
预测模型只使用任务前可获得的信息，输出能耗与风险评估；
仿真环境使用更细粒度的扰动、环境和动力响应生成虚拟观测数据；
验证模块比较两者误差，并通过场景消融分析模型在不同天气条件下的稳定性。
后续接入真实飞行日志或台架实验后，可以继续校准仿真参数和预测模型。
```

## 8. 项目最终定位

最终定位应统一为：

```text
天气驱动的城市低空无人机任务能耗预测与虚拟验证平台
```

关键词：

- 天气驱动
- 任务级预测
- 起飞前评估
- 虚拟验证
- 可量化误差
- 可逐步校准
- 可扩展到高保真仿真和真实日志

# 系统结构

## 目录结构

```text
modules/uav_virtual_validation/
  configs/      城市、世界、场景、天气、飞行器配置
  data/         原始数据、中间数据和处理数据
  docs/         中文文档
  outputs/      城市资产、仿真输出、纠错版本和自动化截图
  scripts/      可执行脚本入口
  src/          Python 包源码
  tests/        测试
  web/          Web 工作台静态页面
```

## 核心链路

```text
城市配置
  -> OSM / Open-Meteo 数据获取
  -> 城市 GeoJSON / 天气场 / 航线
  -> 本地 3D Tiles 转换
  -> Web 工作台加载展示
  -> 人工纠错版本
  -> 修正版 3D Tiles
```

仿真验证链路：

```text
场景配置
  -> 简化仿真器
  -> 虚拟观测时序
  -> 预测结果对比
  -> 指标与报告
```

## 前端职责

`web/index.html` 和 `web/static/app.js` 负责：

- 加载城市资产。
- 控制 Cesium 场景。
- 展示建筑、地面、天气、航线和禁飞区。
- 执行对象选择和属性查看。
- 管理城市要素纠错。
- 调用 `/api/corrections` 保存纠错版本。

前端不负责：

- 重新抓取 OSM。
- 生成 3D Tiles。
- 执行复杂物理仿真。
- 训练或部署能耗预测模型。

## 后端职责

`scripts/serve_web.py` 是轻量本地服务，负责：

- 提供静态页面和输出资产访问。
- 保存城市要素纠错版本。
- 覆盖、删除、设置默认纠错版本。
- 触发修正版城市 3D Tiles 生成。

它不是生产 Web 后端，也不承担用户认证、数据库、多用户并发编辑等能力。

## 自动化检查

自动化检查位于：

```text
modules/web_automation_checks/
```

它使用 Playwright/Chromium 打开页面，检查：

- 页面是否加载成功。
- Cesium 是否出现渲染停止。
- 控制台是否报错。
- 天气点是否可点击。
- 拖拽视角是否生效。
- 本地 3D Tiles 和阴影模式是否正常。

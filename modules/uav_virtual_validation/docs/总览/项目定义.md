# 项目定义

## 一句话定义

本项目是面向城市低空无人机任务的天气驱动能耗预测与虚拟验证平台。

更具体地说：

```text
在任务起飞前，基于航线、载荷、电池参数和可获得天气数据，预测任务能耗、剩余电量和风险边界；在禁飞条件下，使用虚拟城市环境生成可复现观测数据，并对预测结果进行误差评估。
```

## 项目不是什么

- 不是飞控系统。
- 不是无人机底层控制器。
- 不是完整动力学仿真器。
- 不是只用于好看的 3D 城市展示页面。
- 不是用虚拟数据自证模型绝对正确的闭环系统。
- 不是可以直接替代真实试飞的数字孪生最终版。

这些边界必须保持清晰，否则答辩时容易被问题带偏到飞控、姿态控制、电机建模、真实飞行合法性等不属于当前阶段的方向。

## 项目是什么

当前项目由三层组成：

```text
预测层：任务级天气能耗预测模型
验证层：虚拟城市环境、仿真观测、误差计算、场景消融
展示层：Web 工作台、3D 城市、航线、天气、风险与纠错工具
```

核心不是“造一个无人机游戏世界”，而是构建一套可逐步升级的验证链路：

- 输入任务前可获得信息。
- 生成或加载城市低空环境。
- 输出仿真观测数据。
- 对比预测和仿真。
- 留出真实日志校准接口。

## 仿真和部署的区别

答辩中常见质疑：

```text
如果仿真世界也生成电量数据，你们的预测模型是不是重复了？
```

标准回答：

```text
不重复。仿真模型是验证环境，承担“考官”角色；部署侧预测模型是待验证算法，承担“考生”角色。两者输入、复杂度、使用阶段和目标不同。
```

| 对比项 | 虚拟仿真环境 | 部署侧预测模型 |
|---|---|---|
| 角色 | 验证场、观测生成器 | 待验证预测算法 |
| 使用阶段 | 实验验证和场景消融 | 起飞前任务评估 |
| 输入信息 | 高分辨率城市、天气扰动、局部风险、随机变化 | 航线、计划高度、载荷、电池、粗粒度天气 |
| 输出 | 虚拟观测时序、功率、电量、风险状态 | 总能耗、剩余电量、风险评分 |
| 复杂度 | 可以更复杂，不要求部署到机端 | 应轻量、快速、可解释 |
| 目标 | 提供可控对比样本 | 给出任务可达性判断 |

## 当前答辩表述

推荐表述：

```text
在本地禁飞条件下，我们不把虚拟仿真当作真实飞行的绝对替代品，而是把它作为可控实验环境。
预测模型只使用任务前实际可获得的信息；仿真环境使用更细粒度的城市、天气和扰动生成虚拟观测；验证模块比较两者误差，并通过不同天气、城市和航线条件分析模型稳定性。
后续接入真实飞行日志或台架实验后，可以继续校准仿真参数和预测模型。
```

不建议表述：

- “仿真和现实误差极低。”
- “虚拟世界完全等同真实世界。”
- “仿真数据证明模型一定准确。”
- “当前系统已经是完整飞控仿真器。”

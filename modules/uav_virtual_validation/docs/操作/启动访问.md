# 启动访问

## 推荐启动方式

开发板环境优先使用根目录一键脚本：

```bash
./use_scripts/start_uav_virtual_validation.sh
```

如果需要手动启动，可在工作空间根目录运行：


```bash
python3 modules/uav_virtual_validation/scripts/serve_web.py --host 0.0.0.0 --port 8090
```

当前开发板推荐访问地址以脚本输出为准，例如：

```text
http://<开发板IP>:8090/
```

本机访问地址：

```text
http://127.0.0.1:8090/
```

根路径会自动跳转到 Web 页面，因此浏览器直接输入 `<开发板IP>:8090` 即可打开。

## 当前服务进程

服务进程 ID 会写入：

```text
modules/uav_virtual_validation/outputs/web_server.pid
```

日志写入：

```text
modules/uav_virtual_validation/outputs/web_server.log
```

检查当前服务：

```bash
cat modules/uav_virtual_validation/outputs/web_server.pid
ps -p "$(cat modules/uav_virtual_validation/outputs/web_server.pid)" -o pid,cmd --no-headers
```

## 缓存处理

前端静态资源使用 `?v=` 参数控制缓存。

如果页面仍显示旧功能或旧错误：

- 修改 URL 中的 `v=` 参数。
- 使用浏览器无痕窗口。
- 强制刷新页面。

当前推荐版本：

```text
?v=20260521-move1
```

## 常见问题

### 页面拒绝连接

优先检查：

- `serve_web.py` 是否正在运行。
- 端口是否为 `8090`。
- 服务是否绑定 `0.0.0.0`。
- 当前设备是否能访问开发板 IP。

### 页面能打开但资产加载失败

优先检查：

- `outputs/real_city/<城市>/city_summary.json` 是否存在。
- `outputs/real_city/<城市>/tiles/tileset.json` 是否存在。
- 浏览器 URL 是否带了新的 `?v=` 参数。

### 修改了后端但行为没变

`serve_web.py` 是 Python 进程，修改后端代码后需要重启服务。只修改前端 JS/HTML 时通常改 `?v=` 参数即可。

# 城市要素纠错

## 功能目标

城市要素纠错用于修正被误识别为建筑的城市对象。它不是建筑用途分类系统。

典型用途：

- 把误识别为建筑的道路、广场、绿化、水体压低为地面要素。
- 把桥墩、高架支柱等保留为实体障碍。
- 生成修正版本地 3D Tiles，避免原错误建筑继续遮挡修改后的低矮要素。

## 选择方式

- 左键点击建筑：单选。
- 右键点击建筑：加入或移出多选集合。
- `Ctrl/Command + 左键`：加入或移出多选集合。
- 多选后点击“应用纠错”：批量设置当前多选对象。

## 分类体系

当前分类：

| 分类 | 渲染高度策略 | 说明 |
|---|---|---|
| 建筑 | 保留实体高度 | 正常建筑或需要保持障碍高度的对象 |
| 道路/车行道 | 压低为地面要素 | 道路、车道、桥面投影错误时的地面面 |
| 步行区域/人行道 | 压低为地面要素 | 人行区域、步道、铺装路面 |
| 绿化草坪/公园 | 压低为地面要素 | 草地、公园、绿化带 |
| 广场/铺装空地 | 压低为地面要素 | 广场、硬质铺装、空地 |
| 水体 | 压低为地面要素 | 河流、湖面、水池 |
| 停车场 | 压低为地面要素 | 地面停车区域 |
| 施工/裸地 | 压低为地面要素 | 施工区域、裸土、临时空地 |
| 交通设施 | 压低为地面要素 | 地面交通设施、交通岛、低矮附属面 |
| 交通立体结构 | 保留实体高度 | 高架桥、轨道桥、立体道路、桥面结构 |
| 交通支撑结构 | 保留实体高度 | 桥墩、高架桥支柱、高铁轨道支柱 |
| 其他地面要素 | 压低为地面要素 | 无法归入以上类别的低矮地面要素 |
| 疑似误识别/待复核 | 压低为地面要素 | 暂时确认不是建筑但还需复查 |

判断原则：

- 如果对象是无人机需要避障的立体实体，不要压低为地面要素。
- 如果对象是地面铺装、水体、绿化或道路，才归入低矮地面要素。
- 高架桥支柱、桥墩、高铁轨道支柱应优先使用 `交通支撑结构`。

## 纠错显示

当前显示模式：

- 按要素异色：每个分类使用自己的颜色，适合检查分类。
- 地面浅灰：纠错后的低矮要素更接近地面颜色，适合最终展示。
- 柔和淡色：保留分类差异，但降低颜色冲突。
- 低饱和对比：适合长时间人工检查。
- 高对比检查：适合快速发现遗漏或错误分类。
- 仅轮廓标注：尽量减少颜色遮挡，只强调被纠错对象边界。

## 版本管理

版本按城市独立管理。

- `原始版本`：只读，不允许覆盖或删除。
- `保存为新版本`：把当前工作区保存成该城市的新版本。
- `覆盖当前版本`：用当前工作区完整覆盖当前非原始版本。
- `设为默认`：该城市后续加载时优先使用此版本，并把当前“纠错显示”保存为该城市默认显示模式。
- `删除当前版本`：需要连续点击两次确认。

覆盖规则：

```text
覆盖当前版本 = 保存当前工作区的完整状态
```

如果只是新增或修改对象，之前已有记录会保留。如果执行了“清除当前”或“清除多选”，对应记录会从工作区删除，覆盖后也会从版本中删除。

## 未应用提醒

如果已经修改分类下拉框或备注，但还没有点击“应用纠错”，再点击“覆盖当前版本”会弹出提醒。

原因是：表单改动只有在点击“应用纠错”后才写入当前工作区。未应用的表单改动不会进入覆盖保存。

推荐操作：

1. 选择对象。
2. 修改分类或备注。
3. 点击“应用纠错”。
4. 检查显示效果。
5. 点击“覆盖当前版本”或“保存为新版本”。

# 开发板启动

本文档说明如何在当前开发板工作空间启动 `modules/uav_virtual_validation` Web 工作台。

## 当前结论

该子项目可以部署在开发板上提供网页功能。服务器端主要提供静态文件、城市资产和少量纠错版本保存接口；Cesium 三维渲染主要发生在访问者浏览器端，因此通常不需要高性能 GPU 服务器。

开发板需要承担的工作：

- HTTP 服务。
- 读取 `outputs/real_city/` 城市资产。
- 保存城市要素纠错版本。
- 必要时生成修正版 3D Tiles。

客户端浏览器承担的工作：

- 3D 城市渲染。
- 视角交互。
- 图层切换。
- 大部分可视化显示。

## 一键启动

在 `fibocom_slz` 根目录执行：

```bash
./use_scripts/start_uav_virtual_validation.sh
```

脚本会自动：

- 使用 `python3`。
- 绑定 `0.0.0.0`。
- 使用默认端口 `8090`。
- 自动检测开发板 IP。
- 写入 PID 文件和日志文件。

## 访问地址

脚本会输出类似：

```text
本机访问：    http://127.0.0.1:8090/
开发板访问：  http://10.129.169.13:8090/
```

其他电脑需要和开发板处于可访问网络中，然后打开开发板访问地址。

## 常用命令

查看状态：

```bash
./use_scripts/start_uav_virtual_validation.sh status
```

停止服务：

```bash
./use_scripts/start_uav_virtual_validation.sh stop
```

重启服务：

```bash
./use_scripts/start_uav_virtual_validation.sh restart
```

前台调试：

```bash
./use_scripts/start_uav_virtual_validation.sh foreground
```

## 修改端口或 IP

如果 `8090` 被占用：

```bash
UAV_WEB_PORT=8091 ./use_scripts/start_uav_virtual_validation.sh
```

如果需要手动指定开发板 IP：

```bash
UAV_WEB_PUBLIC_IP=10.129.169.13 ./use_scripts/start_uav_virtual_validation.sh
```

## 启动前检查

当前开发板至少需要：

```bash
python3 --version
```

如果需要运行浏览器自动化检查，还需要：

```bash
node --version
npm --version
```

注意：Web 服务本身不依赖 Node.js。

## 资产检查

默认城市和真实城市资产应存在：

```bash
ls modules/uav_virtual_validation/outputs/real_city/manhattan_midtown/city_summary.json
ls modules/uav_virtual_validation/outputs/real_city/manhattan_midtown/tiles/tileset.json
```

如果城市资产缺失，页面可能能打开，但对应城市会加载失败。

## 日志和 PID

日志文件：

```text
modules/uav_virtual_validation/outputs/web_server.log
```

PID 文件：

```text
modules/uav_virtual_validation/outputs/web_server.pid
```

如果启动失败，先查看最近日志：

```bash
tail -n 80 modules/uav_virtual_validation/outputs/web_server.log
```

## 资源压力判断

正常静态服务状态下，服务器端 CPU 占用应接近 0。内存占用通常远低于浏览器端渲染占用。

可能产生明显服务器端压力的操作：

- 保存或覆盖大量城市要素纠错版本。
- 生成修正版 3D Tiles。
- 重新抓取或重建真实城市资产。

普通浏览页面时，主要压力在客户端浏览器。

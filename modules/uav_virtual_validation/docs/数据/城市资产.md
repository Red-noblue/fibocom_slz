# 城市资产

## 数据来源

真实城市资产主要来自：

- OpenStreetMap / Overpass：建筑 footprint、道路、水域、绿地、部分高度字段。
- Open-Meteo：历史天气或预报天气。
- 本项目配置：城市范围、航线、默认高度、天气采样派生规则。

## 输出目录

每个真实城市输出到：

```text
outputs/real_city/<城市名>/
  city_config_snapshot.json
  city_summary.json
  real_buildings.geojson
  ground_layers.geojson
  route.geojson
  real_weather_field.geojson
  historical_weather_open_meteo.json
  tiles/
    tileset.json
    ground_tileset.json
    outline_tileset.json
```

纠错后还会生成：

```text
outputs/real_city/<城市名>/corrected_tiles/<版本>/
```

## 建筑高度规则

建筑高度优先级：

1. OSM `height` 字段。
2. OSM `building:levels` 字段估算。
3. 城市配置中的默认高度。

因此不能宣称所有建筑高度都是真实测量值。更准确的说法是：

```text
建筑 footprint 来自真实地图数据，高度在有 OSM 高度字段时使用真实标注，否则使用层数估算或默认值补全。
```

## 本地 3D Tiles

当前默认使用本地 3D Tiles，而不是浏览器一次性挤出大量 GeoJSON。

优势：

- 大规模城市加载更稳定。
- 建筑、地面、边线可分层控制。
- 纠错版本可以生成修正版 tiles，避免原错误建筑覆盖低矮纠错面。

边界：

- 当前 tiles 仍是项目生成的简化城市几何，不是商业级纹理化城市模型。
- 后续可继续接入更高质量 3D Tiles 或真实影像/地形服务。

## 城市列表维护

城市配置位于：

```text
configs/cities/
```

新增城市时应同时检查：

- 城市配置是否合理。
- OSM 数据是否能正常获取。
- 建筑和地面要素规模是否适合浏览器加载。
- 航线是否穿越异常建筑。
- 天气采样是否覆盖任务区域。
- Web 下拉列表是否补充新城市。

# 天气数据

## 真实源数据

当前天气源主要使用 Open-Meteo。

工程定位：

- 适合城市级天气背景。
- 适合任务级能耗预测输入。
- 适合历史天气回放和情景构造。

不应宣称：

- 可以直接获得建筑尺度 CFD 风场。
- 可以精确描述每栋楼之间的湍流。
- 可以替代现场风速仪和真实飞行日志。

## 分辨率边界

当前可对外说明：

| 类型 | 当前口径 |
|---|---|
| 时间分辨率 | 小时级 |
| 水平空间分辨率 | 约 9-25 km，取决于底层气象产品 |
| 原生高度信息 | 常见为 2m 温湿度、10m/100m 风等气象变量 |
| 城市内细粒度点 | 由项目派生，不是 Open-Meteo 原生建筑尺度观测 |

## 派生天气场

Web 工作台显示的三维天气采样点是项目根据城市范围、航线和高度层派生出来的。

它的作用：

- 给仿真世界提供空间化天气背景。
- 让天气影响在 3D 视图中可检查。
- 支持不同分辨率下的演示和性能权衡。

它不代表真实世界中每个点都有同等精度的实测天气数据。

## 工作台分辨率控制

水平采样：

- 完整 10x6
- 中等 5x3
- 航线走廊 10x2
- 稀疏 4x3

高度采样：

- 全部 5 层
- 低空 30/60/100m
- 任务 100/150/220m

## 答辩建议

推荐表述：

```text
Open-Meteo 提供的是任务级气象背景，不是建筑尺度微气象真值。本项目在仿真侧派生三维天气采样，用于构造可控实验环境；部署侧预测模型只使用任务前可获得的粗粒度天气信息，二者输入边界不同。
```

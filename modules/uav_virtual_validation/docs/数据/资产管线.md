# 资产管线

## 程序化世界

程序化世界配置：

```text
configs/worlds/urban_grid_world.json
```

生成命令：

```bash
python3 modules/uav_virtual_validation/scripts/build_world.py
```

输出：

```text
outputs/urban_grid_world/
  world_summary.json
  world.geojson
  route.czml
```

程序化世界主要用于早期闭环、功能验证和无外部数据依赖的测试。

## 真实城市抓取

单城市抓取示例：

```bash
python3 modules/uav_virtual_validation/scripts/fetch_real_city_assets.py \
  --city modules/uav_virtual_validation/configs/cities/manhattan_midtown.json \
  --output-dir modules/uav_virtual_validation/outputs/real_city/manhattan_midtown
```

该脚本负责获取或生成：

- 建筑 footprint。
- 地面要素。
- 航线。
- 历史天气。
- 三维天气采样点。
- 城市摘要。

## 本地 3D Tiles

生成本地 3D Tiles：

```bash
python3 modules/uav_virtual_validation/scripts/build_local_3d_tiles.py
```

输出目录：

```text
outputs/real_city/<城市名>/tiles/
```

## 城市要素修正管线

人工纠错保存后，`serve_web.py` 会：

1. 保存当前纠错版本 JSON。
2. 根据非建筑地面要素移除对应原建筑。
3. 生成 `corrected_tiles/<版本>/`。
4. 更新 `version_index.json`。
5. 前端重新加载修正版 tiles。

注意：

- `交通立体结构` 和 `交通支撑结构` 不会从建筑 tiles 中移除。
- 低矮地面要素会从建筑 tiles 中移除，避免原错误建筑继续覆盖纠错结果。

## 失败产物处理

如果生成过程失败且产物无意义，应删除失败产物，避免后续页面误加载半成品。

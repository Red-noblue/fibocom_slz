# 路线图

## 阶段 1：本机闭环

目标：跑通预测、仿真、误差对比、Web 展示。

核心产物：

- 程序化世界。
- 简化仿真器。
- 预测对比指标。
- Web 工作台。
- 自动化页面检查。

当前状态：基本完成，并已扩展到真实城市背景。

## 阶段 2：真实城市验证平台

目标：让真实城市背景下的任务验证平台足够可用。

当前已完成：

- 25 个真实城市或片区。
- 本地 3D Tiles。
- 天气采样分辨率控制。
- 城市要素纠错和版本管理。
- 多选批量纠错。
- 视角遥控器。
- 自动化检查。

后续重点：

- 继续修复城市数据异常。
- 增强禁飞区和任务风险配置。
- 增加任务配置和结果播放。
- 把纠错记录和实验结果关联起来。

## 阶段 3：验证分析

目标：从“看城市”升级到“验证模型”。

需要补充：

- 预测曲线和仿真曲线对比。
- 总能耗误差。
- 剩余电量误差。
- 风速、温度、载荷、高度消融。
- 实验报告导出。

## 阶段 4：高保真后端接入

候选方向：

- PX4/Gazebo SITL。
- AirSim。
- Isaac Sim。
- Webots。
- 真实日志 Replay。

原则：

- Web 工作台不绑定单一仿真后端。
- 后端输出统一转换为项目标准 artifacts。
- 高保真仿真用于校准和增强，不替代任务级预测模型。

## 阶段 5：真实校准

目标：用真实日志或台架实验缩小 sim-to-real gap。

需要：

- 真实电池、电流、电压、位置、高度、风速记录。
- 参数辨识流程。
- 校准前后误差对比。
- 可复现实验记录。

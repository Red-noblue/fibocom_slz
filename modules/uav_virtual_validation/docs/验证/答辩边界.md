# 答辩边界

## 最容易被攻击的问题

### 1. 仿真数据是不是自己造出来证明自己？

回答要点：

- 仿真环境不是最终真值，只是禁飞条件下的可控验证环境。
- 预测模型只使用任务前可获得的粗粒度信息。
- 仿真环境可以使用更细粒度扰动，用于生成对比观测。
- 真正的可信度来自误差量化、消融实验和后续真实日志校准。

### 2. 为什么不用真实无人机直接飞？

回答要点：

- 当前本地禁飞，不能进行大规模外场试飞。
- 项目采用分阶段验证路线。
- 当前阶段先建立可复现虚拟验证平台。
- 后续接入台架实验、公开日志或合法真实飞行日志进行校准。

### 3. Open-Meteo 数据够不够精细？

回答要点：

- Open-Meteo 适合任务级天气背景，不是建筑尺度微气象真值。
- 部署侧预测模型本来也无法获得每栋楼之间的高分辨率风场。
- 仿真侧的三维天气采样是派生实验环境，用于测试模型稳健性。

### 4. 当前 3D 城市是不是高保真数字孪生？

回答要点：

- 当前城市使用 OSM footprint 和项目生成的本地 3D Tiles。
- 它足以支撑城市低空场景、航线、遮挡、风险和可视化检查。
- 它不是商业纹理级、毫米级或 CFD 级数字孪生。
- 后续可以接入更高质量 3D Tiles、真实影像、地形和高保真仿真器。

### 5. 城市要素纠错是否主观？

回答要点：

- OSM 数据和自动转换会有误识别，人工纠错是 GIS 数据清洗的常见步骤。
- 纠错版本会保存在本地，支持版本化、默认版本和删除。
- 原始版本只读，便于追溯。
- 纠错目标是修复明显的几何语义错误，不是随意改造数据。

## 推荐最终定位

```text
天气驱动的城市低空无人机任务能耗预测与虚拟验证平台。
```

关键词：

- 天气驱动。
- 任务级预测。
- 起飞前评估。
- 真实城市背景。
- 虚拟验证。
- 可量化误差。
- 可逐步校准。

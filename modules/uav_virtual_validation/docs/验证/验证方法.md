# 验证方法

## 验证对象

本项目验证的是任务级天气能耗预测模型，不验证底层飞控控制精度。

验证问题：

```text
在给定城市、航线、载荷、电池和天气背景下，预测模型对任务能耗、剩余电量和风险边界的估计是否稳定、可解释、可量化？
```

## 基本流程

```text
预测输入
  -> 预测输出
  -> 仿真场景配置
  -> 虚拟仿真执行
  -> 虚拟观测输出
  -> 预测与观测误差对比
  -> 指标、图表和报告
```

## 核心指标

- 总能耗误差。
- 单位距离能耗误差。
- 剩余电量误差。
- 风速变化下的误差变化。
- 温度变化下的误差变化。
- 载荷变化下的误差变化。
- 风险等级一致性。
- 曲线趋势一致性。
- 关键事件触发一致性。

## 实验类型

### 基础实验

固定城市、航线和飞行器参数，只改变天气。

目标：检查模型对天气变化方向是否敏感。

### 城市场景实验

切换不同城市或城市片区。

目标：检查高楼密集、道路密集、开阔区域等环境差异对任务评估的影响。

### 消融实验

分别关闭或固定某些因素：

- 风速。
- 温度。
- 载荷。
- 高度。
- 航线长度。

目标：解释误差来源，而不是只给单个总分。

### 校准实验

后续接入真实日志或台架数据后，比较：

- 仿真观测和真实观测。
- 预测输出和真实观测。
- 校准前后误差变化。

## 工程原则

- 先保证闭环可运行，再提高仿真精度。
- 所有实验输入要可复现。
- 所有误差要可量化。
- 不使用“极低误差”这类无法证明的表述。
- 真实数据接入后再讨论 sim-to-real 误差缩小。

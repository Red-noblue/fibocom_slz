# 命令参考

当前入口脚本：

```bash
./tools/env-bootstrap/bin/env-bootstrap
```

## 基础命令

`doctor <manifest>`

- 检查 manifest、环境文件、目标前缀和 `micromamba` 解析结果。
- 用于确认环境会被建到哪里。

示例：

```bash
./tools/env-bootstrap/bin/env-bootstrap doctor \
  ./tools/env-bootstrap/manifests/uav-modules.env.sh
```

`sync <manifest>`

- 根据 `environment.yml` 创建或更新目标环境。
- 然后把 manifest 中声明的本地包以 editable 方式安装进去。

`run <manifest> -- <command ...>`

- 在目标环境中执行单条命令。
- 团队协作时优先使用这个入口，而不是手工 `activate`。

示例：

```bash
./tools/env-bootstrap/bin/env-bootstrap run \
  ./tools/env-bootstrap/manifests/uav-modules.env.sh -- \
  python --version
```

`freeze-explicit <manifest> <output-file>`

- 导出当前平台的显式 lock 文件。
- 适用于固定平台重建，不适用于跨平台直接复用。

## 共享环境管理命令

`list`

- 扫描 `tools/env-bootstrap/manifests/*.env.sh`
- 扫描仓库根目录 `._envs/`
- 合并输出“已声明环境”和“孤儿环境”

输出字段：

- `NAME`：环境目录名
- `STATE`：`present` / `partial` / `missing`
- `SCOPE`：`shared` / `external` / `orphan`
- `MANAGED BY`：manifest 归属
- `PREFIX`：环境真实路径

示例：

```bash
./tools/env-bootstrap/bin/env-bootstrap list
```

`prune [--yes]`

- 只处理 `._envs/` 下没有任何 manifest 引用的孤儿环境。
- 默认只预览，不删除。
- 加 `--yes` 才会真正删除。

示例：

```bash
./tools/env-bootstrap/bin/env-bootstrap prune
./tools/env-bootstrap/bin/env-bootstrap prune --yes
```

## micromamba 获取

下载顺序：

1. 优先尝试 `micro.mamba.pm`
2. 失败后回退到 `conda-forge current_repodata.json`

这两个都属于在线源切换，不涉及离线环境打包。

## manifest 约定

manifest 是一个 bash 片段，至少需要：

```bash
PROJECT_NAME="uav-modules"
ENV_FILE="../../../modules/uav/environment.yml"
ENV_PREFIX="../../../._envs/uav-modules-py310"
EDITABLES=(
  "../../../modules/uav/uav-weather-energy-predictor"
  "../../../modules/uav/uav-gis-replay-studio"
)
```
